## 测试模块1：login登录（微信授权+游客模式）

### 页面路径：pages/login/index

#### 【P0】登录后未区分游客与已登录用户状态
- **问题描述**: `skipLogin()` 仅执行 `wx.switchTab({ url: '/pages/index/index' })`，游客进入首页后无任何登录引导提示或功能限制提示，用户可能误以为已登录。
- **严重程度**: P0
- **复现步骤**: 1. 打开小程序 2. 点击"暂不登录，先看看" 3. 进入首页 4. 首页展示的用户名为空（未设置姓名），无任何提示引导用户登录

#### 【P1】登录页未处理微信授权按钮真实回调
- **问题描述**: WXML中使用 `<button bindtap="wxLogin">` 而非 `<button open-type="getUserInfo">`，无法获取微信用户头像昵称用于预填充。
- **严重程度**: P1
- **复现步骤**: 1. 点击微信授权登录按钮 2. 触发wx.login获取code 3. 后端返回userInfo可能为空，登录后头像/昵称需用户手动填写
- **涉及文件**: pages/login/index.wxml:24, pages/login/index.js:34-51

#### 【P1】游客模式下可访问全部页面
- **问题描述**: 游客模式下后端未校验token，store.checkLogin()仅在部分页面前端校验，用户可以自由访问匹配、AI对话等需要登录的功能。
- **严重程度**: P1
- **复现步骤**: 1. 不登录 2. 直接通过URL跳转访问/pages/ai/match/index 3. 可正常看到匹配列表（Mock数据）

#### 【P2】登录成功后Toast与页面跳转冗余
- **问题描述**: 登录成功后先showToast再setTimeout跳转，页面切换有明显的1.5秒延迟，用户体验不佳。
- **严重程度**: P2
- **复现步骤**: 1. 点击微信登录 2. 显示"登录成功"Toast 3. 等待1.5秒 4. 跳转首页
- **建议**: 建议直接跳转，或使用更短的延迟

## 测试模块2：index首页（空状态+新手引导）

### 页面路径：pages/index/index

#### 【P0】空状态判定逻辑缺陷
- **问题描述**: `showEmpty` 的判定条件为 `!brochure && (!Array.isArray(recommendData) || recommendData.length === 0)`。即使用户有画册（brochure存在），但recommendList为空时，showEmpty = false，页面既不展示画册也不展示空状态引导，导致空白页。
- **严重程度**: P0
- **复现步骤**: 1. 创建画册 2. MockService.getRecommendList返回空数组 3. 页面只显示用户头部、操作栏和空白卡片区域，下方无任何内容

#### 【P1】新手引导缺失
- **问题描述**: 首次启动或首次使用场景下，没有任何新手引导流程（如蒙层引导、步骤提示），新用户可能不知道如何创建名片。
- **严重程度**: P1
- **复现步骤**: 1. 首次打开小程序并登录 2. 进入首页 3. 页面上有"编辑名片"按钮但无引导指示

#### 【P1】画册封面图片外部URL可能加载失败
- **问题描述**: brochure.cover 使用外部URL（neeko-copilot.bytedance.net），该地址可能跨域或不可达，导致封面图加载失败显示空白。
- **严重程度**: P1
- **复现步骤**: 1. 进入首页 2. 画册卡片区域image标签加载外部URL 3. 如网络不可达则显示空白
- **涉及文件**: pages/index/index.wxml:97, images/default-cover.png兜底在WXML中不存在

#### 【P1】首页数据加载仅在onShow检测dataDirty
- **问题描述**: 首页数据加载完全依赖MockService，没有网络异常的明确兜底文案。"加载失败"的catch中不显示任何Toast。
- **严重程度**: P1
- **复现步骤**: 1. 首次加载 2. Promise.all中某个请求失败 3. catch中只设置loading=false，用户看不到任何反馈

#### 【P2】场景模式切换无实际效果
- **问题描述**: 首页的"场景模式"（personal/business/social）切换后仅更新data变量和显示Toast，没有对页面内容产生任何实质性变化。
- **严重程度**: P2
- **复现步骤**: 1. 展开场景模式 2. 点击"商务对接" 3. Toast提示已切换 4. 页面内容无任何变化
- **建议**: 切换后应该改变推荐列表、展示内容排序或数据过滤逻辑

#### 【P2】升级提示条文案硬编码
- **问题描述**: 升级提示条文案固定为"访客已达{{stats.visitors}}人"，但showUpgradeHint的触发条件是 `visitors >= 3 || recommendData.length >= 3`，包含推荐数量条件但文案只提访客。
- **严重程度**: P2
- **复现步骤**: 1. 当visitors=0但recommendData.length=3时 2. 提示条显示"访客已达0人"，文案矛盾
- **涉及文件**: pages/index/index.js:93, pages/index/index.wxml:35-39

## 测试模块4：ai/chat AI对话（双模式切换）

### 页面路径：pages/ai/chat/index

#### 【P0】AI对话使用`app.isLoggedIn()`但该方法不存在
- **问题描述**: `ai/chat/index.js` 第15行调用 `app.isLoggedIn()`，但App实例上没有定义该方法。store中对应方法为 `store.getState().isLoggedIn`。这会导致JavaScript运行时错误。
- **严重程度**: P0
- **复现步骤**: 1. 进入AI对话页面 2. onLoad中调用app.isLoggedIn() 3. 报错：app.isLoggedIn is not a function
- **涉及文件**: pages/ai/chat/index.js:15

#### 【P0】AI对话发送后使用Mock固定回复
- **问题描述**: send()方法中调用setTimeout直接返回固定文案，无论用户输入什么，AI都回复完全相同的默认消息。没有调用MockService.aiChat()方法。
- **严重程度**: P0
- **复现步骤**: 1. 输入任意问题 2. 点击发送 3. 每次回复均为"你好！我是AI数智名片助手..."，对话无实际意义

#### 【P1】聊天输入框未使用textarea
- **问题描述**: 使用 `<input>` 而非 `<textarea>`，无法换行输入，单行输入体验差。
- **严重程度**: P1
- **复现步骤**: 1. 进入AI对话 2. 尝试输入多行内容 3. input不支持换行
- **涉及文件**: pages/ai/chat/index.wxml:15

#### 【P1】登录蒙层与输入框状态冲突
- **问题描述**: 未登录时显示login-mask蒙层，但输入框的placeholder已变为"请先登录后使用"，底层input仍可聚焦。
- **严重程度**: P1
- **复现步骤**: 1. 未登录状态进入AI对话 2. 可以看到蒙层提示登录 3. 但仍可点击输入框（无障碍问题）

#### 【P2】聊天区域未自动滚动到底部
- **问题描述**: scroll-view设置scroll-into-view="bottom"，但缺少id为"bottom"的锚点view的引用联动，实际不生效。
- **严重程度**: P2
- **复现步骤**: 1. 发送消息 2. 消息条数超过可视区域 3. 不会自动滚动到底部
- **涉及文件**: pages/ai/chat/index.wxml:2,12

## 跨模块问题（适用于刘总视角）

#### 【P1】Mock数据中所有测试图片均使用外部CDN
- **问题描述**: 测试数据和默认头像/封面均使用 `neeko-copilot.bytedance.net` 的外部URL，该域名在企业内网或微信小程序中可能被限制访问。
- **严重程度**: P1
- **涉及模块**: 全部页面（首页、画册、二维码、推荐列表等）

#### 【P1】MVP核心流程（注册→创建名片→分享→匹配→交换）未形成闭环
- **问题描述**: 当前为单页面独立开发状态，页面间仅通过wx.navigateTo跳转而缺乏状态传递。如创建画册后跳转到预览页但仍显示Mock数据。
- **严重程度**: P1
- **复现步骤**: 1. 在画册创建页提交表单 2. 跳转到预览页?id=newId 3. 预览页根据id从MockService获取可能失败 4. 回退到Mock默认画册
