## 测试模块1：profile个人中心（编辑资料）

### 页面路径：pages/profile/profile

#### 【P0】个人中心无法直接编辑资料
- **问题描述**: 个人中心"编辑"按钮跳转到 `/pages/brochure/create/index`（画册创建页面），而非个人资料编辑页。用户无法单独修改头像、姓名、公司、职位等信息。
- **严重程度**: P0
- **复现步骤**: 1. 进入"我的"页面 2. 点击头像旁的"编辑"按钮 3. 跳转到画册创建页面 4. 用户需要重新填写整套表单才能更新基本信息
- **涉及文件**: pages/profile/profile.js:78-79

#### 【P1】访客记录功能标注"功能开发中"
- **问题描述**: "访客记录"菜单项点击后显示Toast"功能开发中"，作为一个核心功能入口不应在正式版本中展示未实现功能。
- **严重程度**: P1
- **复现步骤**: 1. 进入个人中心 2. 点击"访客记录" 3. 显示Toast"功能开发中"
- **涉及文件**: pages/profile/profile.js:92-93

#### 【P1】头像无法点击更换
- **问题描述**: profile页面的头像区域仅用于展示，不可点击更换。用户需要跳到画册创建页才能换头像，路径过深。
- **严重程度**: P1
- **复现步骤**: 1. 进入个人中心 2. 点击头像 3. 无任何反应

#### 【P1】数据概览的"匹配数"始终为0
- **问题描述**: stats.matches 在loadProfile中未被赋值（始终为初始值0），而MockService.getProfile()返回的数据中match_count不为空，但代码中未将其映射到stats.matches。
- **严重程度**: P1
- **复现步骤**: 1. 进入个人中心 2. 查看数据概览 3. 匹配数始终显示为0
- **涉及文件**: pages/profile/profile.js:45-46

#### 【P2】"我的画册"与"编辑"按钮功能重复
- **问题描述**: 顶部"编辑"按钮和菜单中的"我的画册"都跳转到画册创建页面，功能冗余。
- **严重程度**: P2
- **复现步骤**: 1. 点击"编辑"按钮 → brochure/create 2. 返回 3. 点击"我的画册" → brochure/create 4. 两者功能一致

## 测试模块2：qrcode二维码（生成+保存）

### 页面路径：pages/qrcode/index

#### 【P0】二维码未实际生成
- **问题描述**: `generateQRCode()` 方法仅执行setTimeout(800ms)后显示"二维码已就绪"Toast，没有实际使用canvas生成二维码或调用任何二维码生成API。
- **严重程度**: P0
- **复现步骤**: 1. 有名片后进入二维码页面 2. 页面显示"生成中..." 3. 800ms后显示"二维码已就绪" 4. 页面上无任何二维码图片显示
- **涉及文件**: pages/qrcode/index.js:27-35

#### 【P0】saveQR()无实际保存功能
- **问题描述**: `saveQR()` 方法仅显示Toast"长按图片保存"，无实际的canvas转图片或保存到相册功能。
- **严重程度**: P0
- **复现步骤**: 1. 进入二维码页面 2. 点击"保存二维码" 3. 仅显示提示文字，无保存动作

#### 【P1】无名片时直接跳回而非给出可操作的引导
- **问题描述**: onLoad中若brochure为null，Toast提示"请先创建名片"后执行wx.navigateBack()返回上一页，用户体验突兀。
- **严重程度**: P1
- **复现步骤**: 1. 未创建名片的用户点击"我的二维码" 2. 进入页面 3. 立即Toast提示并返回 4. 用户不知道去哪里创建名片
- **建议**: 改为提供引导按钮让用户前往创建

#### 【P1】WXML缺失二维码展示区域
- **问题描述**: qrcode/index.wxml中仅有JS逻辑控制，WXML中缺少canvas或image组件来展示二维码。
- **严重程度**: P1
- **复现步骤**: 1. 检查qrcode/index.wxml 2. 发现没有canvas或image标签
- **注**: 需确认二维码页面的WXML是否存在

## 测试模块3：network/graph信任网络（图谱交互）

### 页面路径：pages/network/graph/index

#### 【P0】Canvas渲染后节点文字溢出无截断
- **问题描述**: 节点名称直接在canvas上用`fillText`绘制，当名称超过2个中文字符时，文字会溢出节点圆形区域。
- **严重程度**: P0
- **复现步骤**: 1. 联系人名称"张三"（2字）勉强显示 2. 名称"赵六"、"孙七"等在18px半径节点上显示不全
- **涉及文件**: pages/network/graph/index.js:238

#### 【P1】图谱动画启动/停止逻辑缺陷
- **问题描述**: `onShow()` 在`animationId`存在时调用`startAnimation()`而非`startAnimation()`存在的逻辑则是调用`renderGraph()`；但正确逻辑应是`onShow`时启动动画循环。当前代码在`onReady`后没有调用`startAnimation()`，导致图谱不会主动动起来。
- **严重程度**: P1
- **复现步骤**: 1. 进入信任网络页面 2. Canvas显示图谱 3. 节点没有任何力学动画 4. 只有在拖拽后才能看到动画

#### 【P1】BFS路径查找功能数据关联不足
- **问题描述**: BFS路径查找需要用户输入"目标用户ID"，但前端没有显示用户ID的地方，用户无法知道应该输入什么ID。
- **严重程度**: P1
- **复现步骤**: 1. 点击"触达路径" 2. 看到输入框要求输入目标用户ID 3. 用户不知道去哪里查找目标ID

#### 【P1】触摸交互命中检测区域为半径的2倍
- **问题描述**: onTouchStart中检测距离时使用 `node.radius * node.radius * 4`（即半径2倍），过大导致点附近空白区域也触发节点拖拽。
- **严重程度**: P1
- **复现步骤**: 1. 在节点附近2倍半径处触摸 2. 节点被拖拽 3. 交互不精准

#### 【P2】"添加联系人"无后端同步
- **问题描述**: addContact仅向本地nodes数组和wx.setStorageSync添加，不与后端同步。刷新页面后本地数据也可能因loadData()覆盖而丢失。
- **严重程度**: P2
- **复现步骤**: 1. 添加联系人"测试" 2. 点击"刷新图谱" 3. loadData被调用 4. 联系人"测试"消失

#### 【P2】图谱图例中紫色节点与页面紫色渐变背景色对比度不足
- **问题描述**: 图谱节点使用紫色渐变(#8b5cf6)，而page-container背景色为#f5f6fa，#8b5cf6在此背景下可读性一般，尤其是小字号文字。
- **严重程度**: P2
- **复现步骤**: 1. 进入信任网络 2. 观察节点与背景的对比度 3. 紫色在浅灰背景上不够突出

## 测试模块4：brochure/preview画册预览（分享+复制）

### 页面路径：pages/brochure/preview/index

#### 【P1】Swiper组件可能因图片加载失败而空白
- **问题描述**: 画册中的封面/头像/公司图片使用外部CDN URL（neeko-copilot.bytedance.net），不可达时swiper页面显示空白image区域。
- **严重程度**: P1
- **复现步骤**: 1. 进入画册预览 2. 部分页面图片加载失败 3. 页面布局破坏

#### 【P1】"分享"功能调用showActionSheet但"生成海报""复制链接"均为占位
- **问题描述**: shareBrochure()展示ActionSheet列出三个选项，后两个点击后仅Toast"功能开发中"。
- **严重程度**: P1
- **复现步骤**: 1. 进入画册预览 2. 点击右上角更多 3. 选择"生成海报" 4. 显示"功能开发中"
- **涉及文件**: pages/brochure/preview/index.js:223-231

#### 【P1】联系方式点击复制后无确认弹窗
- **问题描述**: 电话/邮箱/微信点击后直接复制到剪贴板并Toast"已复制"，用户可能误触导致隐私信息被复制。
- **严重程度**: P1
- **复现步骤**: 1. 画册预览模式下 2. 随意点击联系方式 3. 立即复制到剪贴板 4. 无二次确认
- **建议**: 点击联系方式时应先显示确认弹窗或详情卡片

#### 【P2】翻页指示器(dots)过于简陋
- **问题描述**: 底部dots只是小的圆形view，无页码数字显示，对于较多页数的画册（7+页）不够直观。
- **严重程度**: P2
- **复现步骤**: 1. 进入7页画册 2. 底部显示7个小圆点 3. 无法快速定位到特定页

#### 【P2】空状态提示"立即创建"按钮跳转到create页面
- **问题描述**: 当无画册时，提示"暂无画册内容"并提供"立即创建"按钮跳转到brochure/create，此为用户体验正向设计。
- **严重程度**: P2（正面发现）
- **复现步骤**: 1. 无画册用户进入预览 2. 显示空状态+引导按钮 3. 可跳转到创建页面
