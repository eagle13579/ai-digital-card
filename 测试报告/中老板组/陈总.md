## 四、membership — 会员升级

### 测试路径
个人中心「会员中心」 → 浏览会员等级 → 升级Pro → 确认支付 → 升级成功

### 执行记录

| # | 操作 | 结果 |
|---|------|------|
| 1 | 进入会员中心 | ✅ 显示3个等级卡片（Free/Pro/Enterprise） |
| 2 | Free卡片显示当前等级 | ✅ "已是当前等级" |
| 3 | Pro卡片标记「推荐」标签 | ✅ popular标识 |
| 4 | 查看各等级权益 | ✅ 每个卡片显示权益列表 |
| 5 | 学生优惠卡片 | ✅ 显示Pro版5折优惠信息 |
| 6 | 点击「升级到Pro」 | ✅ 弹窗确认，显示价格和权益 |
| 7 | 确认升级 | ✅ 显示"支付中..." → 1.5秒后提示"升级成功" |
| 8 | 升级后重新进入会员中心 | ✅ 当前等级变为Pro |
| 9 | 点击Enterprise升级 | ✅ 弹窗确认 |

### 问题清单

| # | 页面 | 问题描述 | 严重程度 | 复现步骤 |
|---|------|---------|---------|---------|
| 1 | membership | **getUserProfile()返回值解析错误**：`loadUserLevel`中`const profile = await MockService.getUserProfile() || {}`，但`getUserProfile()`返回`{ data: profile }`，所以`profile.memberLevel`为undefined，始终回退到'free' | **P0** | 进入会员中心 → 即使已升级Pro → 页面仍显示Free |
| 2 | membership | **升级支付完全模拟**：1.5秒setTimeout模拟支付，无真实支付通道对接 | P1 | 确认升级 → 显示"支付中..." → 1.5秒后成功（无真实支付） |
| 3 | membership | **升级成功后store.updateMemberLevel更新但页面无实时反馈**：仅在当前页面更新data，其他页面需要重新加载才能看到会员等级变化 | P2 | 升级Pro → 返回首页 → 首页会员等级仍显示旧等级（需reload） |
| 4 | membership | **学生优惠无认证流程**：显示学生优惠信息但无学生认证入口 | P2 | 会员页 → 学生优惠卡片 → 不可点击 |
| 5 | membership | **等级映射中gold/silver被映射为Pro，但mock数据只用了free/gold/silver**：实际UI只支持free/pro/enterprise三档，gold/silver用户显示"Pro" | P2 | 使用silver用户登录 → 等级显示Pro（可能混淆） |
