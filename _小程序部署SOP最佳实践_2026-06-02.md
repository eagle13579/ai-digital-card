```
□ 拥有微信小程序AppID（路径: 微信公众平台 → 开发 → 开发设置）

□ 微信开发者工具已安装（最新版, 下载地址: https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html）
□ 代码已推送到GitHub（当前分支: master）
□ HTTPS证书有效（当前: Let's Encrypt, 有效期至2026-08-30）
```

### 2.3 域名白名单配置（关键！）

登录 **微信公众平台** → 开发管理 → 开发设置 → 服务器域名：

```
request合法域名:
  https://www.go-aiport.com       # API网关
  https://api.liankebao.com       # 备用API
  
uploadFile合法域名:
  https://www.go-aiport.com
  
downloadFile合法域名:
  https://www.go-aiport.com
  
webview合法域名:
  https://www.go-aiport.com
  https://liankebao.top           # 画册H5分享页
```

> ⚠️ 微信最多可配置20个request域名。配置后需 **5-10分钟** 生效。
> ⚠️ 小程序不支持IP地址作为合法域名，必须使用已备案的域名。

```javascript

api.put('/api/v1/notifications/' + id + '/read')         // PUT简写
```

## §4 提交审核流程

### 4.1 提审前自检清单

```
□ 所有页面正常渲染（在3种机型和基础库版本下测试）
□ API全部使用正式域名（已关闭 urlCheck）
□ 所有图片资源可访问
□ 用户登录/注册流程正常
□ 画册创建/编辑/发布流程完整
□ 二维码生成+保存+分享功能正常
□ 信任网络添加/移除功能正常
□ 无硬编码的测试数据
□ 项目描述和类目已设置
□ 隐私协议已配置（2024年微信新规要求）
```

### 4.2 提交步骤

```
微信开发者工具 → 上传
  → 版本号: 遵循 semver (当前: 1.0.0)
  → 版本描述: 本次变更摘要（如: 新增AI名片画册功能）
  
微信公众平台 → 版本管理 → 提交审核
  → 选择刚上传的版本
  → 填写更新说明（用户可见, 建议简洁）
  → 配置功能页面（勾选画册首页作为功能页）
  → 提交审核
  
审核周期: 通常1-7天（首次审核较长, 更新审核较快）
加急: 每年3次加急机会（紧急修复可用）
```

### 4.3 常见驳回原因及处理

| 驳回原因 | 处理方式 |
|:---------|:---------|
| **类目不匹配** | 在公众平台设置正确类目（建议: 工具 > 效率）|
| **隐私协议缺失** | 在小程序设置中添加隐私协议链接 |
| **界面有未完善功能** | 隐藏或提示「开发中」 |
| **无法体验完整流程** | 提供测试账号在版本描述中 |
| **域名未配置** | 在开发设置中添加request合法域名 |
| **ICP备案问题** | 确保 liankebao.top 已备案 |

## §5 发布上线

### 5.1 全量发布

```
微信公众平台 → 版本管理 → 审核通过版本 → 发布
  → 全量发布（默认, 立即对所有用户生效）
  → 分阶段发布（按比例灰度, 建议大版本更新使用）
  
分阶段发布推荐:
  1% → 观察24h → 10% → 观察24h → 50% → 观察24h → 100%
```

### 5.2 版本回滚

如需回滚到旧版本：

```
微信公众平台 → 版本管理 → 选择历史版本 → 启用
  → 微信会在用户下次打开时自动更新
  → 或在小程序管理后台发起「版本回退」

注意:
  - 回滚仅支持到最近5个版本
  - 回滚后API版本需与小程序版本配套
  - 数据库schema不可回滚（向前兼容）
```

## §6 CI/CD 集成（进阶）

如需将小程序部署纳入GitHub Actions流水线：

### 6.1 使用 miniprogram-ci

微信官方提供命令行CI工具 `miniprogram-ci`：

```bash
npm install miniprogram-ci --save-dev
```

### 6.2 GitHub Actions 配置示例

```yaml
# .github/workflows/miniapp-deploy.yml
name: 小程序上传
on:
  push:
    branches: [release/miniapp]

jobs:
  upload:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm install miniprogram-ci
      - name: 上传小程序
        run: |
          npx miniprogram-ci upload \
            --pp ./liankebao-miniapp \
            --pkp ${{ secrets.MINIAPP_PRIVATE_KEY }} \
            --appid ${{ secrets.MINIAPP_APPID }} \
            --uv 1.0.${{ github.run_number }} \
            --desc "自动构建 ${{ github.sha }}"
```

> 私钥获取: 微信公众平台 → 开发 -> 开发设置 → 小程序代码上传密钥

## §7 常见问题FAQ

| 问题 | 排查步骤 |
|:-----|:---------|
| **页面空白** | ①检查app.json中pages路径正确 ②检查控制台有无报错 ③清缓存重编译 |
| **API请求失败** | ①检查小程序开发工具中urlCheck是否为false ②检查域名白名单 ③检查API服务是否运行 |
| **登录态丢失** | ①检查api.js中token处理逻辑 ②检查wx.getStorageSync('token')是否正常 |
| **画册数据不显示** | ①检查API返回格式 ②检查前端数据绑定字段名是否匹配 |
| **二维码无法保存** | ①检查用户授权 ②检查downloadFile域名白名单 ③iOS需用户主动触发 |
| **构建上传失败** | ①检查project.config.json中appid ②检查密钥是否过期 |
| **真机调试与开发工具表现不一致** | ①检查基础库版本 ②检查机型和微信版本兼容性 |
