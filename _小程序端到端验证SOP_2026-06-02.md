## §2 验证API连通性

### 2.1 在开发者工具中检查

```
开发者工具 → 调试器 → Console（控制台）
输入:
  wx.request({ url: 'https://liankebao.top/api/v1/health', success: r => console.log(r.data) })

预期输出:
  {status: "ok", service: "AI数字名片 v2.2", version: "2.2.0", ...}
```

### 2.2 如果报错

| 报错 | 原因 | 处理 |
|:-----|:------|:------|
| `errno:600001` | 测试号不支持https | 改用真机测试或配置AppID |
| `request:fail` | 域名未配置白名单 | 开发工具：详情 → 本地设置 → 勾选「不校验合法域名」 |
| statusCode:503 | 服务未启动 | 服务器运行: `systemctl start brochure-api.service` |

### 2.3 快捷键

```
开发工具编译: Ctrl + S（保存文件后自动热重载）
打开调试器: Ctrl + Shift + I
真机调试: 工具栏 → 真机调试（需AppID）
清除缓存: 工具栏 → 清除 → 全部清除
```

## §5 画册编辑流程（6步）— 核心功能

### 步骤

```
1. 在画册首页点击「编辑画册」
2. 按顺序填写6步:
```

#### Step 1: 基本信息

```
  ☐ 姓名: （必填, 如: 张三）
  ☐ 公司: （必填, 如: 测试科技有限公司）
  ☐ 职位: （如: CEO）
  ☐ 个人简介: （选填, 如: 专注于AI技术）
  → 点击「下一步」
```

#### Step 2: 联系方式

```
  ☐ 手机: （必填, 如: 13800138000）
  ☐ 微信: （选填）
  ☐ 邮箱: （选填）
  → 点击「下一步」
```

#### Step 3: 产品列表

```
  ☐ 点击「+ 添加产品」
  ☐ 填写产品名称、价格
  ☐ 可添加多个产品
  → 点击「下一步」
```

#### Step 4: 供需需求

```
  ☐ 选择类型: 供应 / 需求
  ☐ 标题: （必填）
  ☐ 描述: （必填）
  → 点击「下一步」
```

#### Step 5: 信任网络

```
  ☐ 可搜索用户添加信任（选填）
  → 点击「下一步」
```

#### Step 6: 预览发布

```
  ☐ 预览画册效果
  ☐ 点击「发布画册」
```

### 预期结果

```
☐ 显示「保存成功」提示
☐ 自动返回画册首页
☐ 画册首页现在显示刚刚填写的信息
```

### 常见问题

| 问题 | 处理 |
|:-----|:------|
| 点击下一步没反应 | 检查必填项是否已填（姓名+公司+手机+标题+描述）|
| 发布后没返回 | 检查控制台有无报错，截图给我 |
| 提示「请先登录」 | 返回「我的」页面重新登录 |
