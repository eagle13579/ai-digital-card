## 四、架构决策

```
链客宝主站（会员+支付+用户）
    ↕ 链客宝桥接（已有 chainke_bridge.py）
AI数字名片服务（画册+匹配+标签）
    ↕ API
微信小程序（前端展示+填单+分享）
```

**会员同步策略**：小程序端注册/登录 → 调用AI数字名片API → AI名片通过chainke_bridge同步到链客宝主库
**数据库策略**：AI名片独立数据库（已有），通过桥接层同步到主库。不双写，主库为权威来源。
**付费解锁策略**：match_records新增contact_unlocked字段，金牌用户每月配额解锁。
