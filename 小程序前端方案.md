## 1. 设计原则

### 1.1 品牌调性
- **主色**：紫蓝渐变 `#667eea → #764ba2`（延续Alpine.js SPA的太空紫风格）
- **辅色**：玫瑰粉 `#f5576c`（CTA按钮、高亮标识）
- **背景**：深空渐变 `#0f0c29 → #302b63 → #24243e`
- **字体**：系统默认（-apple-system, PingFang SC）
- **圆角**：大圆角 `12px-20px`，毛玻璃效果 `backdrop-filter: blur(12px)`

### 1.2 微信小程序规范适配
- 使用 `navigationStyle: custom` 自定义导航栏，维持品牌一致性
- TabBar 3项：首页、匹配、我的
- 针对小程序特性：使用 `wx.getUserProfile` / `wx.login` 做微信一键登录
- 使用 `wx.cloud` 或CDN处理图片上传

### 1.3 技术栈建议
- 基于现有 `liankebao-miniapp` 原生框架开发（WXML+JS+WXSS），或迁移至 `liankebao-weapp`（Taro + React）
- 建议采用 **Taro 3.x**（React语法）复用现有 `liankebao-weapp` 的架构，实现跨端潜力
- 翻页引擎：小程序端使用 `wx.createSelectorQuery` + CSS3 3D Transform模拟翻页，轻量级无需额外库；或用 `taro-flip` 等适配组件

## 2. 小程序页面结构

共6个核心页面 + 1个WebView降级页：

| 编号 | 页面名称 | 路由路径 | 类型 | 对应现有资产 |
|------|----------|----------|------|-------------|
| 1 | 首页（名片展示/电子画册） | `/pages/index/index` | Tab页 | Alpine.js `brochure_viewer.html` + 现有 `brochure/index` |
| 2 | 填单页（信息填写+图片上传） | `/pages/card-editor/index` | 主流程 | Alpine.js `card_editor.html` + 现有 `brochure-editor/index` |
| 3 | 画册预览页（翻页H5查看） | `/pages/brochure-preview/index` | 主流程 | Alpine.js `card_editor` Step 2 预览 |
| 4 | 匹配结果页 | `/pages/match/index` | Tab页 | Alpine.js `card_editor` Step 3 匹配 |
| 5 | 我的页面 | `/pages/mine/index` | Tab页 | 现有 `mine/index` |
| 6 | 登录页 | `/pages/login/index` | 前置 | Alpine.js `index.html` 登录模块 |
| — | WebView降级（备用） | `/pages/webview/index` | 辅助 | 现有 `webview/index` |

### 页面间传参说明

| 跳转 | 参数 | 说明 |
|------|------|------|
| 首页 → 填单页 | `?editId={brochure_id}` | 编辑已有名片时传ID |
| 填单页 → 预览页 | `?data={JSON}` | 传递填写的表单数据（通过Storage） |

#### 关键交互

1. **翻页**：`StPageFlip` 3D翻页引擎，模拟纸质画册翻页效果
2. **上滑**：可进入完整详情模式（展开更多字段）
3. **分享**：调用 `wx.shareAppMessage` 生成小程序卡片
4. **联系我**：触发匹配流程，若对方未授权则提示"查看联系方式需发起匹配请求"
5. **编辑**：跳转至填单页（编辑模式）
6. **状态判断**：如果用户未创建名片，显示空状态引导 → "创建我的第一张名片"

#### 数据来源
- 自己查看：调用 `/api/v1/brochures/mine`（需token）
- 他人查看：通过分享链接携带 `share_token`，调用 `/api/brochures/share/{share_token}`

- 小程序无法直接使用DOM操作的StPageFlip库

- **推荐**：方案B + Taro组件封装，或使用 `taro-flip` 社区组件

### 6.2 Token管理策略

```
登录/注册 → 获取 access_token
  → wx.setStorageSync('token', token)
  → 全局请求拦截器自动注入 Authorization header
  → 401响应 → 清除token → 跳转登录页
```

### 6.3 请求封装（建议）

```javascript
// utils/request.js
const BASE_URL = 'https://api.lianke-bao.com'

function request(method, url, data, needAuth = true) {
  return new Promise((resolve, reject) => {
    const header = { 'Content-Type': 'application/json' }
    if (needAuth) {
      const token = wx.getStorageSync('token')
      if (token) header['Authorization'] = 'Bearer ' + token
    }
    wx.request({
      url: BASE_URL + url,
      method,
      data,
      header,
      success: (res) => {
        if (res.statusCode === 401) {
          wx.removeStorageSync('token')
          wx.reLaunch({ url: '/pages/login/index' })
          reject(new Error('未登录'))
        } else if (res.data.code === 0) {
          resolve(res.data)
        } else {
          reject(new Error(res.data.message || '请求失败'))
        }
      },
      fail: reject
    })
  })
}
```

**缺点**：
- StPageFlip需要手写翻页组件（CSS 3D Transform）
- 代码复用度低（和Alpine.js/React不共享）
- 组件化方案较原始（template + slot）

**推荐度**：⭐⭐⭐⭐（快速上线首选）

### 方案B：Taro 3.x（React语法）

**优点**：
- 与 `liankebao-weapp` 架构一致，可复用React组件
- 支持跨端（H5/小程序/RN）
- React生态丰富（状态管理、Hooks）

**缺点**：
- 现有 `liankebao-miniapp` 原生页面需迁移
- 翻页组件需找Taro适配版
- 编译调试流程多一层

**推荐度**：⭐⭐⭐（长期维护推荐）

### 推荐选择

> **短期（1-2周MVP）**：方案A原生小程序，快速验证核心流程
> **长期（1月+）**：迁移至方案B Taro，保持与 `liankebao-weapp` 统一架构
