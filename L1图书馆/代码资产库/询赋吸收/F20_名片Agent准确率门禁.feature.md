# F20 — 名片Agent准确率门禁

## 心智模型（Mental Model）

90.2%不是天花板，而是地板——这是所有Agent上线必须通过的最低准确率基准。就像建筑师验收时的"最小荷载标准"：低于这个标准，不予通过。准确率门禁是自动化质量防线，防止低质量Agent进入生产环境。

## 原子映射

| 字段 | 内容 |
|------|------|
| **原子ID** | A13 |
| **原子名称** | 90.2%准确率基准 |
| **Feature名称** | 名片Agent准确率门禁 |
| **优先级** | P1（质量保障） |

## 核心功能

以90.2%为基线基准，所有Agent输出必须达到或超过此准确率方可上线：

- **基线设定**：90.2%为初始基线，定期更新
- **自动评估**：准确率自动计算（LM-as-Judge + 人工抽检校准）
- **CI/CD门禁**：准确率指标纳入CI/CD流水线
- **自动阻断**：低于阈值的Agent版本自动拒绝发布
- **基线更新**：月/季度基线校准（基于历史数据和业务需求）
- **异常分析**：准确率下降时自动分析根因

## 适用场景

- 所有Agent的CI/CD发布门禁
- Agent版本迭代的自动化质量检查
- 第三方模型/API接入的验收基准
- 灰度发布前后的质量对比

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 发布阻断触发率 | >90%（当准确率低于阈值时） |
| 基线更新频率 | 月/季 |
| 误阻断率 | <5%（准确率达标但被误判未达标） |
| 门禁检查耗时 | <30min（完整评估流水线） |

## 技术栈

- **评估引擎**: LM-as-Judge（复用F18评估看板）
- **CI/CD集成**: GitHub Actions / Jenkins插件
- **基准管理**: 版本化的基准配置文件
- **仪表盘**: Grafana + 门禁状态看板
- **通知**: 企业微信/钉钉/Slack自动通知

## 吸收建议

| 询赋实现 | AI数智名片适配方案 |
|----------|-------------------|
| 硬性90.2% | 分层门禁（基础Agent 90.2%，高价值Agent 95%） |
| 统一基准 | 按Agent类型差异化基准 |
| 仅阻断 | 阻断+改进建议（指出哪个维度不达标） |
| 静态基准 | 动态基准（随模型能力提升逐步提高） |
