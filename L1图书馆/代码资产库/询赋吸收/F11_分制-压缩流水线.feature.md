# F11 — 分制-压缩流水线

## 心智模型（Mental Model）

"少即是多"——Agent的输出不是越详细越好，而是越精炼越好。在每一层Agent输出后强制压缩：提取关键信息、丢弃噪音、生成紧凑摘要。类似采访编辑：原始素材数万字，最终稿件数百字。压缩是系统吞吐量的倍增器。

## 核心功能

所有Agent输出前执行"压缩"步骤，从原始输出中提取关键信息，丢弃噪音，生成紧凑摘要：

- **压缩率可配置**：默认压缩率90%（即输出缩小到原始10%）
- **Lossless模式**：无损压缩（仅去除格式冗余，保留全部信息）
- **Lossy模式**：有损压缩（提取摘要，丢弃细节）
- **压缩策略**：按重要性评分保留Top-K信息
- **链式压缩**：多级压缩流水线（粗→细→精）

## 适用场景

- 大客户报告生成（多Agent输出合并压缩为简洁报告）
- 六度人脉路径摘要（图谱路径压缩为关键节点链）
- 数据分析结果精简（数字统计压缩为关键指标）
- 跨Agent知识汇总（多来源信息合并摘要）

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 压缩率 | >=90% |
| 信息保留率 | >=98% |
| 压缩耗时 | <原始输出的20% |
| 用户满意度 | >95%（压缩后信息的可用性） |

## 技术栈

- **压缩器**: LLM摘要 + 规则提取双通道
- **压缩策略**: 可插拔策略引擎（按内容类型选择最优策略）
- **质量校验**: LM-as-Judge验证压缩后信息完整度
- **存储**: 压缩结果缓存（避免重复压缩）
