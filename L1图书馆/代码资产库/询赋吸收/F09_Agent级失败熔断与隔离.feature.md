# F09 — Agent级失败熔断与隔离

## 心智模型（Mental Model）

单个Agent的失败不应像多米诺骨牌一样传播到整个系统。借鉴电路熔断器（Circuit Breaker）模式：当检测到连续失败时断开连接，保护下游系统不被淹没；经过冷却期后尝试恢复。每个Agent拥有独立的失败边界。

## 核心功能

单个Agent调用的失败不会级联传播，提供完整的熔断保护机制：

- **熔断器模式**：Closed（正常）→ Open（断开）→ Half-Open（半开试探）状态机
- **超时阈值**：可配置的超时时间，超时即触发熔断
- **重试策略**：指数退避重试（Exponential Backoff），最大重试次数可配
- **降级响应**：熔断时返回优雅降级响应（兜底数据 / 提示信息）
- **死信队列**：失败的请求进入死信队列，异步重试或人工介入
- **失败事件通知**：支持Webhook/事件总线回调

## 适用场景

- 数智名片6个Agent（knowledge_agent, data_agent, qa_agent等）的稳定运行
- 外部API调用（如数据库查询、第三方服务）的容错
- 批量任务处理中的单任务失败隔离
- 多Agent并发执行的错误边界

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 熔断响应时间 | <500ms |
| 级联失败率 | =0%（完全隔离） |
| 降级覆盖率 | >90% |
| 误熔断率 | <1% |
| 自动恢复成功率 | >95% |

## 技术栈

- **熔断器**: 自定义Circuit Breaker实现（参考Hystrix思路）
- **死信队列**: Redis List / 本地内存队列
- **监控**: Prometheus指标暴露
- **配置**: 动态配置中心（支持运行时调整阈值）

## 吸收建议

| 询赋实现 | AI数智名片适配方案 |
