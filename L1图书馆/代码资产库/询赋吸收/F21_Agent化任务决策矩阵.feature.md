# F21 — Agent化任务决策矩阵

## 心智模型（Mental Model）

不是所有任务都适合用Agent解决。把任务画在矩阵上：横轴=读取密集型 vs 写入密集型，纵轴=可并行 vs 需耦合。只有"读取+可并行"这个象限的任务才推荐Agent化。就像工具箱里的工具——锤子很好，但不是所有问题都是钉子。

## 核心功能

定义任务分类矩阵，帮助判断任务是否适合Agent化：

- **任务分类矩阵**：
  - 象限I（推荐Agent化）：读取密集型 + 可并行
  - 象限II（谨慎Agent化）：读取密集型 + 需耦合
  - 象限III（不推荐Agent化）：写入密集型 + 可并行
  - 象限IV（不推荐Agent化）：写入密集型 + 需耦合
- **Agent化可行性评分**：0-100分，基于多维评估（读取/写入比、并行度、耦合度）
- **决策建议**：是否Agent化 + 推荐架构模式
- **历史决策追踪**：记录所有Agent化决策及结果

## 适用场景

- 新功能开发时的技术决策（要不要用Agent？）
- 架构评审（现有功能是否适合Agent化改造？）
- Agent化改造评估（遗留系统迁移的成本效益分析）
- 技术方案评审标准化

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 决策准确率 | >90%（事后验证） |
| Agent化改造成功率 | 提升30% |
| 评审耗时 | 减少50% |
| 决策追溯率 | 100%（所有决策可查） |

## 技术栈

- **决策引擎**: 规则引擎 + LLM辅助评估
- **任务分析器**: 自动分析任务特征（读取量、写入量、并行潜力）
- **评分模型**: 加权评分卡（可配置权重）
- **知识库**: 历史决策案例库（辅助新决策）
