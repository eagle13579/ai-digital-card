# F18 — Agent质量评估看板

## 心智模型（Mental Model）

"用AI评估AI"——LM-as-Judge：用更高阶的模型来评估Agent的输出质量。就像学校用更资深的老师来批改新老师的试卷。自动化评估流水线保证每个Agent版本的质量可量化、可追溯。

## 原子映射

| 字段 | 内容 |
|------|------|
| **原子ID** | A11 |
| **原子名称** | LM-as-Judge评估 |
| **Feature名称** | Agent质量评估看板 |
| **优先级** | P1（质量保障） |

## 核心功能

自动化评估流水线，覆盖所有Agent：

- **评测样本**：至少20条评测样本，覆盖典型场景和边界情况
- **高阶模型评估**：使用GPT-4/Claude等更强大的模型评估Agent输出
- **人类抽检校准**：人类专家抽检 >=10%的评估结果
- **质量维度**：准确性、完整性、有用性、安全性、格式合规
- **质量看板**：可视化展示所有Agent的质量评分趋势
- **回归检测**：版本变更自动触发回归测试

## 适用场景

- Feature验收（上线前质量确认）
- 灰度发布前后的质量门禁
- Agent回归测试（版本迭代的持续质量保障）
- 第三方模型/API的质量监控

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 评估覆盖 | 所有Agent |
| 人类抽检率 | >=10% |
| 评估耗时 | <30min（全Agent流水线） |
| 评估-人类一致性 | >90%（LM评估与人类评估的一致率） |

## 技术栈

- **评估引擎**: LLM API（高阶模型作为Judge）
- **样本管理**: YAML/JSON定义评测集
- **流水线编排**: CI/CD集成（GitHub Actions / Jenkins）
- **结果存储**: 时序数据库 + 可视化看板
- **对比分析**: 版本间质量差异分析

## 吸收建议

| 询赋实现 | AI数智名片适配方案 |
|----------|-------------------|
| 固定评测集 | 动态评测集（根据线上bad case自动扩充） |
| 单模型Judge | 多Judge委员会（多个模型投票） |
| 简单评分 | 多维度评分雷达图 |
| 无趋势分析 | 质量趋势线+异常预警 |

## 可复用程度

- **LM-as-Judge评估模型**: 100% 可复用
- **评测流水线设计**: 100% 可复用
- **具体实现**: 75% 可复用（需适配本地Agent接口）
