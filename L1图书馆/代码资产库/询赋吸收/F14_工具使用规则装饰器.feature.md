# F14 — 工具使用规则装饰器

## 心智模型（Mental Model）

工具不是万能的，每个工具有适用的前提条件和边界。就像用户手册上的"使用须知"——操作前必须检查前置条件，操作后必须验证后置条件。工具装饰器将使用规则附着在工具描述上，让Agent在调用工具前"先读说明书"。

## 核心功能

每个Agent工具（数据库查询、API调用、文件操作等）的描述中加入使用规则，工具定义标准化：

- **前置条件**：调用前必须满足的条件（如：用户已认证、参数非空）
- **后置条件**：调用后保证的状态（如：缓存已更新、日志已记录）
- **边界情况**：极端输入的处理方式（空值、超大值、特殊字符）
- **错误处理**：各类错误的降级方案和提示信息
- **使用示例**：1-2个典型调用示例
- **成本声明**：工具调用的预估Token/时间成本

## 适用场景

- knowledge_agent的知识库检索工具
- data_agent的数据查询工具
- growth_agent的营销工具
- 任何需要外部资源访问的Agent工具

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 工具误用率 | <5% |
| 工具选择准确率 | >95% |
| 工具描述标准化覆盖率 | 100% |
| 错误处理覆盖率 | >90% |

## 技术栈

- **装饰器模式**: Python装饰器 / Java注解风格
- **工具注册表**: 集中式工具注册与检索
- **Schema定义**: JSON Schema描述工具参数和返回值
- **运行时校验**: 调用前自动校验前置条件

## 吸收建议

| 询赋实现 | AI数智名片适配方案 |
|----------|-------------------|
| 手动规则 | 半自动规则模板（从历史调用模式提取） |
| 静态规则 | 动态规则（可运行时调整） |
| 单次校验 | 链式校验（前置→执行→后置） |
| 无统计 | 工具使用统计仪表盘（成功率、误用率） |
