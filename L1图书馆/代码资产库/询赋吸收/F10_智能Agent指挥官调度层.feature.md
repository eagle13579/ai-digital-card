# F10 — 智能Agent指挥官调度层

## 心智模型（Mental Model）

指挥官（Commander）负责任务规划与分发，工人（Worker）只负责执行和压缩结果。指挥官维护全局任务DAG（有向无环图），确保依赖关系正确；工人聚焦单一职责，不关心全局。类似工厂流水线的工长制：工长安排工作，工人只管干活。

## 核心功能

基于现有 agent_runtime.py 的升级：一个指挥官Agent负责任务规划与分发，6个工人Agent并行执行：

- **指挥官Agent**：接收用户请求 → 任务拆解 → DAG规划 → 分发子任务 → 结果合并
- **工人Agent**：knowledge_agent, data_agent, qa_agent, support_agent, backend_agent, architecture_agent
- **任务DAG管理**：支持串行/并行/条件分支依赖
- **结果合并器**：多工人输出按Schema对齐合并
- **动态资源分配**：根据任务优先级和工人负载动态调度

## 适用场景

- 用户请求 → 指挥官拆解 → 分发到多个工人Agent → 合并结果 → 返回用户
- 复杂业务流程（人脉搜索+名片匹配+信任度评估）的自动编排
- 跨Agent知识融合查询
- 多步骤报告生成流水线

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 任务分发延迟 | <200ms |
| 工人并行度 | >=4（可同时运行） |
| 指挥官决策准确率 | >95% |
| DAG规划耗时 | <任务总时间5% |
| 合并结果完整度 | 100%（无遗漏子任务） |

## 技术栈

- **现有资产**: `agent_runtime.py`
- **指挥官**: LLM+规则混合决策引擎
- **DAG引擎**: 轻量级DAG运行时（参考Airflow思路但轻量化）
- **通信**: 内存级IPC（进程内多线程）/ Redis（跨进程）
- **结果合并**: JSON Schema驱动的合并器

## 吸收建议
