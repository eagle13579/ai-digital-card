# F19 — Token消耗分析仪表盘

## 心智模型（Mental Model）

"Token就是货币"——在1亿人级的AI产品中，Token消耗直接等于运营成本。不测量就无法管理。实时/历史的Token消耗分析让每一分钱花得明明白白：哪个Agent最耗Token？哪个用户用量最大？哪个时段成本最高？

## 原子映射

| 字段 | 内容 |
|------|------|
| **原子ID** | A12 |
| **原子名称** | 系统级Scaling Law |
| **Feature名称** | Token消耗分析仪表盘 |
| **优先级** | P1（成本控制） |

## 核心功能

实时/历史Token消耗分析，多维度可视化：

- **按Agent维度**：每个Agent（knowledge/qa/data/support等）的Token消耗占比
- **按用户维度**：单用户/用户分组的Token消耗排名
- **按时段维度**：日/周/月/季度的消耗趋势
- **预算预警**：设定预算上限，接近时自动告警
- **异常检测**：检测突增/突降的Token消耗模式
- **成本换算**：Token → 金额的实时换算

## 适用场景

- 每周成本审查（运营团队）
- Agent效率优化（开发团队查找高消耗Agent）
- 异常调用检测（发现滥用/异常流量）
- 用户分级计费（免费层/付费层用量追踪）

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 数据延迟 | <5min |
| 异常检测召回率 | >95% |
| 预算预警准确率 | >99% |
| 多维度查询响应 | <3s |

## 技术栈

- **数据采集**: Agent Runtime埋点（拦截器模式）
- **存储**: 时序数据库（InfluxDB / ClickHouse）
- **可视化**: Grafana / 自建Dashboard
- **预警**: Prometheus AlertManager
- **分析**: SQL + 预聚合（物化视图）

## 吸收建议

| 询赋实现 | AI数智名片适配方案 |
|----------|-------------------|
| 单维度统计 | 多维度下钻（Agent→用户→Session→单次调用） |
| 批量延迟数据 | 实时流式采集 |

| 只看消耗 | 消耗+效益对比（Token消耗 vs 用户满意度） |
