# F13 — Token预算指令系统

## 心智模型（Mental Model）

Token就是货币——每次Agent调用都在消耗真金白银。每条指令必须有明确的"预算"：输出长度限制、格式约束、终止条件。超出预算自动截断或触发降级。类似企业差旅政策：出差有报销上限，超出自费。

## 原子映射

| 字段 | 内容 |
|------|------|
| **原子ID** | A06 + A07 |
| **原子名称** | 指令具体化规则 + 工具启发式装饰 |
| **Feature名称** | Token预算指令系统 |
| **优先级** | P1（成本控制） |

## 核心功能

每条Agent指令附带Token预算上限，指令必须有明确的约束：

- **输出长度限制**：最大/最小Token数约束
- **输出格式约束**：JSON/YAML/表格等格式要求
- **终止条件**：何时停止生成（如：找到答案即停）
- **超出预算自动截断**：强制截断或触发降级
- **降级策略**：缩短回复、减少检索深度、使用更便宜的模型
- **成本监控**：实时记录每条指令的Token消耗

## 适用场景

- 所有Agent调用，尤其高频调用（用户名片查询、人脉推荐）
- 面向1亿人级的成本控制核心
- 免费层用户（Tier限制）的调用控制
- 批量处理任务的Token总量预算

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 单次调用Token消耗降低 | >40% |
| 指令模糊度评分 | <5%（由LM-as-Judge评估） |
| 预算超标率 | <2% |
| 降级触发对用户感知影响 | <5%可感知 |

## 技术栈

- **预算控制器**: 拦截器模式（AOP注入Agent调用前后）
- **Token计数器**: tiktoken / 模型对应tokenizer
- **降级管理器**: 策略模式（不同降级方案可插拔）
- **监控**: Prometheus + Grafana仪表盘
- **配置**: 动态配置中心（支持按用户等级/时段调整预算）

## 吸收建议

| 询赋实现 | AI数智名片适配方案 |
|----------|-------------------|
| 固定预算 | 动态预算（按用户等级/任务复杂度调整） |
| 简单截断 | 智能降级（优先缩短而非截断） |
| 无监控 | 实时Token仪表盘+预算预警 |
| 统一预算 | 分Agent预算池（knowledge_agent vs qa_agent差异化） |

## 可复用程度

- **预算控制模型**: 100% 可复用
- **降级策略设计**: 100% 可复用
- **具体实现**: 75% 可复用（需适配本地模型tokenizer）
