# F15 — 渐进式人脉搜索

## 心智模型（Mental Model）

"先广后精"的搜索策略——第一步像撒网捕鱼，获取尽可能多的相关人脉（广度）；第二步像钓鱼，在第一步的结果集中精确定位目标（深度）。避免一开始就钻进单一关系链的误区。

## 核心功能

搜索人脉分两阶段：

- **第一阶段（广度搜索）**：搜索关联人脉网络，获取尽可能多的相关节点和关系。使用宽泛的搜索条件，不限制关系深度。
- **第二阶段（深度搜索）**：在第一阶段结果集中深度挖掘，聚焦精确匹配。使用细化条件，深入分析关系链。
- **阶段过渡条件**：可配置（何时从广度切换到深度）
- **渐进式结果展示**：先展示概览，再展示详情

## 适用场景

- 六度人脉查找（通过多级关系找到目标人脉）
- 行业人脉挖掘（先找行业共性的多节点，再精确目标）
- 潜在客户推荐（先推荐潜在客户集合，再评分排序）
- 知识图谱的搜索与导航

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 第一阶段召回率 | >95% |
| 第二阶段精确率 | >85% |
| 搜索耗时 | <3s |
| 用户满意度 | >90% |

## 技术栈

- **图数据库**: Neo4j / Nebula Graph（六度关系存储）
- **搜索策略**: BFS（广度优先）→ DFS（深度优先）切换
- **相关性评分**: PageRank / Personalized PageRank
- **缓存**: 第一阶段结果缓存（同用户重复搜索加速）

## 吸收建议

| 询赋实现 | AI数智名片适配方案 |
|----------|-------------------|
| 固定两阶段 | 自适应阶段数（根据结果密度动态决定） |
| 简单切换条件 | 智能切换（信息增益下降时自动切换） |
| 无缓存 | 语义缓存（相似搜索复用阶段一结果） |
| 通用搜索 | 个性化搜索（基于用户历史行为调整权重） |
