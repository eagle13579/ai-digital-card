# F17 — Agent灰度发布平台

## 心智模型（Mental Model）

新版本Agent不要一刀切全量上线——先让小部分用户试用，收集反馈，确认没问题后再逐步扩大到全部用户。就像桥梁施工后的负载测试：先过一个人，再过一群人，最后通车。1亿人级的产品，一次全量事故就是灾难。

## 核心功能

新版本Agent与旧版本共存，按用户分组/流量比例灰度切换：

- **灰度策略**：按用户ID哈希、按用户等级、按地域、随机比例
- **A/B对比**：新老版本同期对比（自动收集质量指标）
- **逐步放量**：支持阶梯式放量（10% → 25% → 50% → 100%）
- **一键回滚**：发现问题立即全量回滚
- **灰度看板**：实时展示灰度状态、质量指标、用户反馈
- **自动放量**：质量指标达标后自动进入下一阶段

## 适用场景

- Agent Prompt升级（模板修改后灰度验证）
- 模型切换（GPT-4 → 新模型）
- 工具逻辑更新（知识库检索算法变更）
- 新增Agent的试运行

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 灰度切换时间 | <1min |
| 回滚时间 | <30s |
| 发布事故率降低 | 80% |
| 灰度期间用户感知差异 | <5%可感知 |

## 技术栈

- **配置中心**: Consul / etcd（灰度策略动态下发）
- **流量路由**: API Gateway（Nginx/Kong）+ 策略匹配
- **指标采集**: Prometheus + 自定义Agent质量指标
- **自动化**: 灰度流水线集成到CI/CD
- **看板**: Grafana仪表盘

## 吸收建议

| 询赋实现 | AI数智名片适配方案 |
|----------|-------------------|
| 单一灰度策略 | 多维度灰度（用户ID+地域+设备混合策略） |
| 手动放量 | 自动放量（指标达标自动升级） |
| 简单回滚 | 智能回滚（自动识别问题版本） |
| 无A/B对比 | 完整A/B实验平台（含统计显著性检验） |
