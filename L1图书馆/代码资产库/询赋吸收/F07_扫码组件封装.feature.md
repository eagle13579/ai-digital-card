# F07 — 扫码组件封装

## 心智模型（Mental Model）

扫码是线下社交转线上的**物理入口**。用户通过扫描对方二维码，完成"看到名片 → 扫码 → 查看资料 → 发起建联"的完整闭环。

询赋使用 `html5-qrcode` 库实现浏览器端的二维码扫描，适用于 H5 页面。在微信小程序环境中，需替换为小程序原生 `wx.scanCode` API。

## Architecture Decision Records

### ADR-017: 扫码组件封装为独立可复用组件

| 项目 | 决策 |
|------|------|
| 决策 | 扫码功能封装为独立组件/页面，与业务逻辑解耦 |
| 理由 | 1) 扫码逻辑可在多个入口复用（首页扫码、搜索框扫码）；2) 替换扫码库只需改组件内部 |
| 后果 | 需定义清晰的输入输出接口 |

### ADR-018: 扫码结果为加密编码的 userId

| 项目 | 决策 |
|------|------|
| 上下文 | 二维码内容包含用户标识，需防止被恶意遍历 |
| 决策 | 二维码编码为 `xunfu://user/{encryptedUserId}` 格式 |
| 理由 | 协议前缀便于识别；加密防止 ID 序号猜测 |
| 后果 | 前端需增加解码逻辑；后端需提供解码接口 |

## 组件接口设计（抽象）

```typescript
interface ScanResult {
  type: 'user' | 'platform' | 'resource';
  id: string;
  raw: string;          // 原始扫码内容
}

interface ScanComponentProps {
  onScan: (result: ScanResult) => void;
  onError?: (error: Error) => void;
  continuous?: boolean;  // 是否连续扫码
}
```

## 扫码流程

```
用户扫码
  → 识别协议前缀
    → xunfu://user/{id}    → 跳转用户名片页
    → xunfu://platform/{id} → 跳转平台详情页
    → xunfu://resource/{id} → 跳转资源详情页
  → 无前缀 → 视为普通文本搜索
```

## 微信小程序适配方案

```javascript
// 替换 html5-qrcode → 微信原生 API
Page({
  scanCode() {
    wx.scanCode({
      onlyFromCamera: true,
      scanType: ['qrCode'],
      success: (res) => {
        const result = this.parseScanResult(res.result);
        if (result) {
          this.handleScanResult(result);
        }
      },
      fail: (err) => {
        wx.showToast({ title: '扫码失败', icon: 'none' });
      }
    });
  },

  // 协议解析
  parseScanResult(raw: string): ScanResult | null {
    const patterns = {
      user: /^xunfu:\/\/user\/(.+)$/,
      platform: /^xunfu:\/\/platform\/(.+)$/,
      resource: /^xunfu:\/\/resource\/(.+)$/,
    };
    for (const [type, pattern] of Object.entries(patterns)) {
      const match = raw.match(pattern);
      if (match) return { type, id: match[1], raw };
    }
    return null;
  }
});
```

## 核心要点总结

| 要点 | 说明 |
|------|------|
| 入口 | 小程序首页、名片页、搜索页均可触发扫码 |
| 输出 | 扫码结果统一为 `{ type, id, raw }` 结构 |
| 协议 | `xunfu://{type}/{id}` 格式，预留扩展性 |
| 容错 | 非协议格式内容视为关键词搜索 |
| 权限 | 扫码后若目标用户设置了 privacy 限制，需按 F04 可见性矩阵做二次校验 |

## 吸收建议

| 询赋实现 | AI数智名片适配方案 |
|----------|-------------------|
| html5-qrcode (H5) | wx.scanCode (原生) |
| 自定协议 URL Scheme | 沿用类似协议 `bizcard://user/{id}` |
| 加密 userId | 建议 base64 编码即可，不想增加复杂度可不加密 |
| 组件封装 | 小程序中封装为通用工具方法 + 页面 |

## 可复用程度

- **协议设计**: 100% 可复用
- **扫码流程设计**: 100% 可复用
- **具体实现代码**: 30% 需替换为小程序 API
