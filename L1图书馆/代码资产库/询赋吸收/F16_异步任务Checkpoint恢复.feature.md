# F16 — 异步任务Checkpoint恢复

## 心智模型（Mental Model）

长跑运动员不是一口气跑完全程——每公里有补给站记录状态。如果中途摔倒，可以从最近的补给站重新出发而不是从头开始。Checkpoint就是Agent任务的"补给站"：每个步骤持久化状态，失败时从最近Checkpoint恢复。

## 核心功能

异步Agent任务（数据清洗、批量处理、报告生成）每个步骤持久化状态，失败后从最近Checkpoint恢复：

- **自动Checkpoint**：每个步骤完成后自动保存状态
- **手动Checkpoint**：支持人工设定检查点
- **断点恢复**：失败后自动定位最近有效Checkpoint
- **Checkpoint查看**：管理界面查看所有Checkpoint
- **手动触发恢复**：支持管理员手动选择恢复点
- **Checkpoint清理**：过期Checkpoint自动清理策略

## 适用场景

- 批量名片信息采集（数千张名片分析）
- 人脉图谱重建（大规模图数据处理）
- 日/周报生成（长流程数据聚合+生成）
- 数据清洗与ETL管道

## 关键指标

| 指标 | 目标值 |
|------|--------|
| 恢复成功率 | >99% |
| 平均恢复时间 | <5s |
| Checkpoint存储开销 | <总存储10% |
| 数据零丢失 | 已持久化的步骤100%不丢失 |

## 技术栈

- **状态持久化**: SQLite / LevelDB（本地）/ PostgreSQL（分布式）
- **任务引擎**: Celery / 自定义异步任务队列
- **恢复管理器**: 检查点定位+状态重建逻辑
- **清理策略**: TTL-based + 容量阈值混合策略
