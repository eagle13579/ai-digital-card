# F02 — 平台角色管理体系

## 心智模型（Mental Model）

平台是一个**带有治理结构的商务社群**。每个平台有三级角色，构成金字塔式治理体系：

```
秘书长 (secretary_general) — 创始人/超级管理员，1个平台1位
  │
秘书处 (secretariat)       — 运营管理团队，可有多位
  │
会员 (member)              — 普通参与者，可有多位
```

平台的核心价值是**资源聚合 + 数据报告**。秘书长可以通过商业报告看到平台的资源分布、成员贡献排名、城市/行业覆盖度。

## Architecture Decision Records

### ADR-004: 角色通过 `platform_members` 关联表表示，而非用户在平台表上的字段

| 项目 | 决策 |
|------|------|
| 上下文 | 用户可加入多个平台，不同平台角色可能不同 |
| 决策 | 使用 `platform_members` 关联表存储 `(platform_id, user_id, role)` |
| 理由 | 多对多关系的标准做法；支持用户跨平台担任不同角色 |
| 后果 | 查询用户角色需 JOIN 关联表 |

### ADR-005: 创建者自动成为秘书长

| 项目 | 决策 |
|------|------|
| 上下文 | 创建平台时需要明确谁是负责人 |
| 决策 | create() 方法在 INSERT platforms 后，自动 INSERT platform_members 并设置 role='secretary_general' |
| 理由 | 保证数据完整性，避免创建者无角色的状态 |
| 后果 | 不支持邀请制创建，创建即管理 |

### ADR-006: 商业报告使用聚合查询实时计算

| 项目 | 决策 |
|------|------|
| 上下文 | 平台管理需要成员统计、资源排名、地区/行业覆盖 |
| 决策 | getManagementData() 使用3条 SQL 聚合查询实时返回数据 |
| 理由 | 数据量级不大（单平台<1万条），实时查询足够快且无需维护物化视图 |
| 后果 | 大型平台需考虑缓存或预计算 |

## 核心代码提取

```typescript
// 角色排序：秘书长 > 秘书处 > 会员
ORDER BY 
  CASE pm.role 
    WHEN 'secretary_general' THEN 1 
    WHEN 'secretariat' THEN 2 
    ELSE 3 
  END,
  pm.joined_at DESC
```

```typescript
// 商业报告数据聚合
static getManagementData(platformId: string) {
  // 1. 角色分布统计
  SELECT pm.role, COUNT(*) as count FROM platform_members WHERE platform_id = ? GROUP BY pm.role

  // 2. 资源排名 TOP10
  SELECT u.id, u.nickname, COUNT(r.id) as resource_count
  FROM platform_members pm
  JOIN users u ON u.id = pm.user_id
  LEFT JOIN resources r ON r.owner_id = u.id AND r.platform_id = pm.platform_id
  WHERE pm.platform_id = ?
  GROUP BY u.id ORDER BY resource_count DESC LIMIT 10

  // 3. 城市覆盖度
  SELECT COUNT(DISTINCT city) as count FROM resources WHERE platform_id = ? AND city != ''

  // 4. 行业覆盖度
  SELECT COUNT(DISTINCT industry) as count FROM resources WHERE platform_id = ? AND industry != ''
}
```

## 数据库 Schema 映射

```sql
CREATE TABLE platforms (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  platform_no TEXT UNIQUE,        -- 平台编号
  creator_id TEXT REFERENCES users(id),
  annual_fee REAL DEFAULT 0,      -- 年费
  description TEXT DEFAULT '',
  created_at DEFAULT unixepoch(),
  updated_at DEFAULT unixepoch()
);

CREATE TABLE platform_members (
  id TEXT PRIMARY KEY,
  platform_id TEXT REFERENCES platforms(id),
  user_id TEXT REFERENCES users(id),
  role TEXT CHECK(role IN ('secretary_general', 'secretariat', 'member')),
  joined_at DEFAULT unixepoch(),
  UNIQUE(platform_id, user_id)
);
```
