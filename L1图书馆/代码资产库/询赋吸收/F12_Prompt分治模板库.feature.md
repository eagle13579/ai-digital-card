# F12 — Prompt分治模板库

## 心智模型（Mental Model）

单一庞大Prompt难以维护和优化——将其拆分为职责单一的小模板：输入解析模板只负责理解输入，信息提取模板只负责提取数据，格式化输出模板只负责构造回复。类似微服务的设计哲学：每个模板做好一件事，模板之间通过组合实现复杂功能。

## 核心功能

每个Agent的Prompt拆分为职责单一的模板，模板可组合可继承：

- **输入解析模板**：理解用户输入、提取关键参数、识别意图
- **信息提取模板**：从知识库/数据库中提取所需数据
- **分析推理模板**：对提取信息进行逻辑推理和加工
- **格式化输出模板**：按指定格式构造最终回复
- **质量控制模板**：检查输出是否满足质量标准，不合格则重做

## 适用场景

- 所有Agent的Prompt管理与迭代
- 新增Agent的快速生成（组合现有模板）
- Prompt版本管理与A/B测试
- Agent行为统一升级（改一个模板影响所有关联Agent）

## 技术栈

- **模板引擎**: Jinja2 / Handlebars风格模板语法
- **模板注册表**: YAML格式的模板定义+元数据
- **组合器**: 流水线式模板执行顺序编排
- **版本管理**: Git-based模板版本控制
- **测试框架**: 每个模板配独立测试用例

## 吸收建议

| 询赋实现 | AI数智名片适配方案 |
|----------|-------------------|
| 静态模板 | 动态渲染模板（可注入运行时变量） |
| 单层模板 | 分层模板（全局模板→领域模板→Agent专属模板） |
| 无版本管理 | 模板版本化+灰度切换 |
