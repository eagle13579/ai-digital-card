# 员工入职/离职安全SOP

> 版本: v1.0 | SOC2 REF: DOC-09
> 创建日期: 2026-07-10

## 入职流程
1. 签署保密协议(NDA)
2. 分配最小必要权限
3. 开通GitHub/飞书/服务器访问
4. 完成安全培训(参见DOC-05)
5. 设置MFA(如有)
6. 分配开发环境

## 离职流程
1. 立即撤销所有系统访问
2. 回收SSH Key/API Token
3. 回收GitHub/飞书权限
4. 回收服务器/数据库访问
5. 交接文档归档
6. 30天后清理个人数据

## 权限审计
- 每月审计活跃用户列表
- 每季度审计权限合理性
- 离职权限必须在24h内回收

## 责任矩阵
| 环节 | 责任人 | 时限 |
|------|--------|:----:|
| 权限开通 | 技术负责人 | 入职当天 |
| 安全培训 | 安全负责人 | 入职1周内 |
| 权限回收 | 技术负责人 | 离职通知后2h内 |
| 数据清理 | 运维负责人 | 离职后30天 |
