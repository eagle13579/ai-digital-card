# 渗透测试模板

## 1. 测试范围

### 1.1 API 端点
| 模块 | 端点 | 方法 |
|------|------|------|
| 认证 | `/api/v1/auth/*` | POST/GET |
| 用户 | `/api/v1/user/*` | GET/PUT/DELETE |
| 名片 | `/api/v1/brochure/*` | CRUD |
| GraphQL | `/graphql` | POST |
| 公开 API | `/api/v1/public/*` | GET |
| Webhook | `/api/v1/webhooks/*` | POST/PUT |

### 1.2 Web 应用
- 前端 SPA (React/Vue)
- SSR 渲染页面（SEO）
- 静态资产（CDN + S3）

### 1.3 基础设施
- 反向代理 / 负载均衡
- 数据库（只读访问测试）
- Redis 缓存
- 对象存储（桶权限）

## 2. 方法论：OWASP Top 10 (2021)

| 编号 | 漏洞类别 | 测试用例示例 | 期望结果 |
|------|----------|--------------|----------|
| A01 | 失效的访问控制 | 越权访问其他用户数据 | 返回 403 |
| A02 | 加密失效 | TLS 降级攻击 | 拒绝连接 |
| A03 | 注入 | SQL/NoSQL/OS注入 | 参数化查询防御 |
| A04 | 不安全的设计 | 密码重置 Token 可预测 | Token 随机性验证 |
| A05 | 安全配置错误 | 默认凭据/目录遍历 | 无默认凭据 |
| A06 | 易受攻击的组件 | 已知 CVE 版本扫描 | 依赖项无已知漏洞 |
| A07 | 认证和会话管理失败 | JWT 签名绕过 | 验证签名算法 |
| A08 | 软件和数据完整性失效 | CI/CD 供应链攻击 | 签名校验 |
| A09 | 安全日志和监控失效 | 审计日志绕过 | 强制日志记录 |
| A10 | SSRF | 内网地址探测 | URL 白名单过滤 |

## 4. 报告模板

### 4.1 漏洞条目格式

```
## [严重度] 漏洞标题

- **ID:** PT-2026-001
- **类别:** [OWASP 分类]
- **端点:** [受影响的 URL/端点]
- **测试日期:** YYYY-MM-DD
- **发现人:** [测试人员]

### 复现步骤
1. 请求: `curl -X <METHOD> <URL> -H "<header>" -d '<payload>'`
2. 预期行为:
3. 实际行为:

### 影响
[描述潜在安全影响]

### CVSS 评分
- 基础分: X.X
- 向量: CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:N/A:N

### 修复建议
[具体修复方案]

### 状态
- [ ] 已确认
- [ ] 已修复
- [ ] 已验证关闭
```

### 4.2 严重度定义

| 严重度 | CVSS 范围 | 定义 | SLA |
|--------|-----------|------|-----|
| 严重 | 9.0–10.0 | 远程代码执行 / 未授权数据访问 | 24h 修复 |
| 高危 | 7.0–8.9 | 越权 / 敏感信息泄露 | 72h 修复 |
| 中危 | 4.0–6.9 | XSS / CSRF / 信息泄露 | 7 天修复 |
| 低危 | 0.1–3.9 | 信息枚举 / 配置问题 | 30 天修复 |
| 信息 | N/A | 技术发现 | 无 SLA |
