### 1.2 数据流方向

```
小程序登录 → 获取 code → AI名片后端 /api/auth/wx-login
  → 链客宝桥接层同步会员等级 → 返回 JWT (含会员等级信息)
  → 小程序缓存 token + 会员信息

匹配浏览 → GET /api/match/engine
  → 返回脱敏用户列表 (不包含 phone/wechat)
  → 免费用户可见匹配但不可见联系方式

付费解锁 → POST /api/unlock/contact
  → 检查配额 → 扣对接券 → 返回真实联系方式
  → 配额不足 → 跳转支付 → 链客宝支付 → 回调 → 更新配额
```

## 2. 技术栈选型

| 层级 | 技术 | 理由 |
|------|------|------|
| **前端框架** | UniApp (Vue 3 + Vite) | 一套代码编译到微信小程序 + H5，降低维护成本 |
| **前端状态管理** | Pinia | Vue 3 官方推荐，比 Vuex 更轻量 |

| **后端ORM** | SQLAlchemy 2.0 (已有) | 保持现有模型体系 |
| **数据库** | SQLite (dev) → PostgreSQL (prod) | 已有配置兼容 |
| **小程序云** | 微信云开发 (可选) | 图片托管/云函数降级方案 |

| **支付** | 链客宝已有微信支付路由 | 复用 /api/payment/wxpay/unified-order |

### 4.2 匹配结果 API 返回脱敏联系方式的设计

**核心原则**：不管用户是什么等级，API 返回的匹配数据中 `phone` 和 `wechat` 字段**始终脱敏**。前端通过 `/api/unlock/contact` 单独获取真实联系方式。

**脱敏规则**：

```python
def mask_phone(phone: str) -> str:
    """手机号脱敏: 138****1234"""
    if not phone or len(phone) < 7:
        return ""
    return phone[:3] + "****" + phone[-4:]

def mask_wechat(wechat: str) -> str:
    """微信号脱敏: wx****789"""
    if not wechat or len(wechat) < 4:
        return ""
    return wechat[:2] + "****" + wechat[-3:]
```

**匹配结果返回格式**（修改 `match.py` 路由）：

```json
{
    "matches": [
        {
            "user_id": 42,
            "user_name": "张三",
            "user_company": "某某科技",
            "user_title": "CEO",
            "user_avatar": "https://...",
            "score": 0.85,
            "common_tags": [...],
            "phone": "138****1234",      // ← 始终脱敏
            "wechat": "wx****789",       // ← 始终脱敏
            "is_trusted": true,          // ← 是否在信任网络中
            "has_unlocked": false        // ← 当前用户是否已解锁该用户
        }
    ]
}
```

## 5. 图片处理流水线

### 5.1 架构设计

```
用户上传 → 小程序 chooseImage → uni.uploadFile
  → POST /api/upload/image
    → 接收原始图片 (multipart/form-data)
    → 校验文件类型 (jpg/png/webp)
    → 校验文件大小 (max 10MB)
    → 临时存储原图
    → 压缩流水线:
        ├─ 1. 检查是否 > 1200px 宽 → 缩放到 1200px (保持比例)
        ├─ 2. 检查质量 > 80 → 调整质量到 80
        ├─ 3. 转换为 WebP 格式
        ├─ 4. 生成多分辨率:
        │     ├─ thumbnail: 200x200 (头像/列表缩略图)
        │     ├─ medium: 600x auto (详情页)
        │     └─ original: 1200x auto (原始浏览)
        └─ 5. 返回三种尺寸 URL
    → 存储到 uploads/{user_id}/{uuid}/
    → 返回 JSON { urls: { thumbnail, medium, original } }
```

### 5.2 技术方案

**Python 后端实现 (Pillow + 可选 Sharp)**：

```python
from PIL import Image
import io, os, uuid

class ImagePipeline:
    SIZES = {
        "thumbnail": (200, 200),   # 正方形裁剪
        "medium": (600, 0),        # 宽600，高度自适应
        "original": (1200, 0),     # 宽1200，高度自适应
    }

    @staticmethod
    def process(file_bytes: bytes, user_id: int) -> dict:
        img = Image.open(io.BytesIO(file_bytes))

        # 统一转 RGB (兼容 RGBA PNG)
        if img.mode in ("RGBA", "P"):
            img = img.convert("RGB")

        base_dir = f"uploads/{user_id}/{uuid.uuid4().hex}"
        os.makedirs(base_dir, exist_ok=True)

        urls = {}
        for name, (w, h) in ImagePipeline.SIZES.items():
            copy = img.copy()
            if w > 0 and copy.width > w:
                ratio = w / copy.width
                new_h = int(copy.height * ratio)
                copy = copy.resize((w, new_h), Image.LANCZOS)

            if h > 0:
                # 居中裁剪
                copy = ImagePipeline.center_crop(copy, w, h)

            path = f"{base_dir}/{name}.webp"
            copy.save(path, "WEBP", quality=80, optimize=True)
            urls[name] = f"/{path}"

        return urls
```

## 6. 会员身份打通方案

### 6.1 流程设计

```
小程序启动
    │
    ▼
wx.login() → 获取 code
    │
    ▼
POST /api/auth/wx-login { code }
    │

    │     ├── 查找/创建 users 表记录（用 openid）
    │     └── 调用链客宝桥接层：
    │           ├── POST /api/external/brochure/user-sync
    │           │     → 链客宝返回 membership_tier
    │           └── 同步到本地 users.membership_tier
    │
    ▼
返回 JWT（含 sub=user_id）+ 用户信息（含 membership_tier）
    │
    ▼
小程序缓存 token + 会员等级 → 控制 UI 展示
```

### 6.2 链客宝桥接增强

修改 `chainke_bridge.py`，新增用户映射方法：

```python
async def sync_user_from_chainke(openid: str, phone: str = "") -> Optional[dict]:
    """
    从链客宝同步用户信息和会员等级。

    流程：
    1. 用 openid 或 phone 在链客宝查找用户
    2. 创建映射关系（如果不存在）
    3. 返回会员等级信息

    Returns:
        {
            "chainke_user_id": 123,
            "membership_tier": "gold",
            "membership_expires_at": "2026-12-31T23:59:59",
            "match_credits": 20
        }
    """
    api_base = _get_api_base()
    if not api_base:
        # 无链客宝时使用本地默认值
        return {"membership_tier": "free", "match_credits": 3}

    url = f"{api_base.rstrip('/')}/api/external/brochure/user-sync"
    payload = {"openid": openid, "phone": phone, "source": "mini_program"}
    resp = _http_request("POST", url, data=payload)
    if resp is None:
        return {"membership_tier": "free", "match_credits": 3}
    return resp.get("data", resp)
```

### 6.3 Token 含会员信息

修改 JWT payload 追加会员字段：

```python
def create_access_token(user: User) -> str:
    to_encode = {
        "sub": str(user.id),
        "membership_tier": user.membership_tier or "free",
        "role": user.role,
        "exp": datetime.utcnow() + timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES),
    }
    return jwt.encode(to_encode, settings.JWT_SECRET, algorithm=settings.ALGORITHM)
```

## 7. 付费解锁流程

### 7.1 完整流程

```
用户在匹配列表看到某人 → 点击"解锁联系方式"
    │
    ▼
检查 match_credits > 0 ?
    │
    ├── 是 → POST /api/unlock/contact { target_user_id }
    │          ├── 检查是否已解锁过 → 直接返回
    │          ├── 扣除 1 张对接券
    │          ├── 记录 QuotaLog
    │          ├── 创建 UnlockRecord
    │          └── 返回真实 phone + wechat
    │
    └── 否 → 提示 "对接券不足"
              │
              ▼
    支付页面 → 选择购买数量 (1张/5张/10张)
              │
              ▼
    POST /api/payment/wxpay/unified-order
              │
              ├── 创建 PaymentOrder (本地)
              ├── 桥接 → 链客宝 /api/payment/wxpay/unified-order
              │           (复用链客宝的微信支付配置)
              └── 返回 payment_params (预支付参数)
              │
              ▼
    小程序端 wx.requestPayment(payment_params)
              │
              ▼
    支付成功 → 微信回调 → /api/payment/wxpay/callback
              │
              ▼
    更新 PaymentOrder.status = "paid"
    增加用户 match_credits
    返回小程序 → 刷新配额 → 继续解锁
```

### 7.2 关键约束

| 规则 | 实现 |
|------|------|
| 免费会员每月 3 张对接券 | 每月 1 号自动重置（或首次登录时） |
| 金卡会员每月 20 张 | 购买会员时一次到账，按月重置 |
| 同一用户不可重复扣费 | UnlockRecord 防重表 (user_id + target_user_id 联合唯一) |
| 解锁后可重复查看 | 解锁后永久有效，不重复扣费 |
| 对接券消耗日志 | 写入 QuotaLog，运营可追溯 |

### 7.3 UnlockRecord 新增表

```python
class UnlockRecord(Base):
    __tablename__ = "unlock_records"

    id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)
    user_id: Mapped[int] = mapped_column(Integer, ForeignKey("users.id"))
    target_user_id: Mapped[int] = mapped_column(Integer, ForeignKey("users.id"))
    credits_spent: Mapped[int] = mapped_column(Integer, default=1)
    unlocked_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now())

    # 联合唯一约束
    __table_args__ = (
        UniqueConstraint("user_id", "target_user_id", name="uq_user_unlock_target"),
    )
```

## 附录 A：关键设计决策

### A.1 为什么不在小程序直接调微信支付，而是通过后端桥接？

- 链客宝已有完整的微信支付配置（AppID、Secret、商户号、证书）
- 复用链客宝支付路由意味着支付回调统一由链客宝处理
- 小程序只获取预支付参数，不直接管理密钥

### A.2 为什么脱敏在 API 层做而不是前端？

- 安全原则：数据在传输链路上越晚脱敏越安全
- 如果前端实现脱敏，抓包工具可以直接拿到真实数据
- API 层统一脱敏，前端无论什么组件都拿不到真实联系方式

### A.3 会员数据以哪里为准？

- **链客宝主库**是会员等级的权威数据源
- AI 名片后端本地缓存会员等级（含过期时间）
- 每次 wx-login 同步，支付回调强制同步
- 如果链客宝不可达，降级到本地缓存（最多允许 30 分钟延迟）

### A.4 对接券是按照自然月重置还是购买月？

- 免费会员：自然月重置（每月 1 号 0 点）
- 付费会员：购买日的每月对日重置
- 重置脚本：QuotaLog 记录 + cron 定时任务
