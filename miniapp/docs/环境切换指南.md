# AI数智名片 - 环境切换指南

## 概述

本项目支持在 Mock 数据模式和真实 API 模式之间快速切换，方便开发和测试。

## 快速切换

### 切换到 Mock 模式（默认）

修改 `utils/mockService.js` 文件，将 `USE_MOCK` 设置为 `true`：

```javascript
const MockService = {
  USE_MOCK: true,  // Mock模式
  // ...
}
```

### 切换到真实 API 模式

修改 `utils/mockService.js` 文件，将 `USE_MOCK` 设置为 `false`：

```javascript
const MockService = {
  USE_MOCK: false,  // 真实API模式
  // ...
}
```

## 模式对比

| 特性 | Mock模式 | 真实API模式 |
|------|---------|-----------|
| 数据来源 | 本地模拟数据 | 后端服务器 |
| 网络请求 | 无 | 有 |
| 后端依赖 | 无需启动 | 必须启动 |
| 开发效率 | 高（无需等待接口） | 低（需等待接口响应） |
| 数据真实性 | 模拟数据 | 真实数据 |
| 适用场景 | 前端开发、功能验证 | 联调测试、线上环境 |

## 真实API模式配置

### 后端服务要求

1. 后端服务必须运行在 `http://192.168.7.48:8201`
2. 确保后端服务已启动且正常运行
3. 确保防火墙允许8201端口入站

### API基础URL配置

API基础URL在 `utils/request.js` 中配置：

```javascript
const DEV_IP = '192.168.7.48'   // 后端服务器IP
const DEV_PORT = '8201'          // AI数智名片后端端口
```

如需修改服务器地址，修改上述常量即可。

### 微信开发者工具配置

1. 打开微信开发者工具
2. 点击工具栏 → 详情 → 本地设置
3. 勾选"不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书"
4. 确保基础库版本选择合适的版本（建议3.16.x）

## API接口列表

项目使用的API接口如下：

### 认证模块
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/logout` - 用户登出

### 用户模块
- `GET /api/users/me` - 获取当前用户信息
- `PUT /api/users/me` - 更新用户信息

### 画册(名片)模块
- `GET /api/brochures` - 获取画册列表
- `GET /api/brochures/{id}` - 获取画册详情
- `POST /api/brochures` - 创建画册
- `PUT /api/brochures/{id}` - 更新画册

### 小程序专用模块
- `POST /api/v1/miniapp/login` - 小程序登录
- `GET /api/v1/miniapp/cards` - 获取名片列表
- `GET /api/v1/miniapp/qrcode` - 获取小程序码

### 其他模块
- `GET /api/tags` - 获取标签列表
- `GET /api/match/recommend` - 获取推荐列表
- `GET /api/trust/network` - 获取信任网络
- `GET /api/visitors/{id}/stats` - 获取访客统计
- `POST /api/ai/write` - AI写作
- `POST /api/ai/generate` - AI生成

## 切换流程

### 从Mock切换到真实API

1. 修改 `utils/mockService.js`：`USE_MOCK: false`
2. 启动后端服务
3. 检查服务器IP和端口是否正确
4. 重新编译小程序
5. 在控制台查看网络请求日志

### 从真实API切换回Mock

1. 修改 `utils/mockService.js`：`USE_MOCK: true`
2. 重新编译小程序
3. 无需启动后端服务即可正常运行

## 常见问题

### Q: 切换到真实API模式后，小程序报timeout错误？

**原因**：后端服务未启动或网络不通

**解决方法**：
1. 检查后端服务是否已启动
2. 检查服务器IP和端口是否正确
3. 检查防火墙是否允许8201端口入站
4. 在浏览器中访问 `http://192.168.7.48:8201` 确认服务是否可访问

### Q: 切换到Mock模式后，数据不更新？

**原因**：缓存问题

**解决方法**：
1. 在微信开发者工具中清除缓存（工具栏 → 详情 → 清除缓存）
2. 重新编译小程序

### Q: 真实API返回的数据格式与Mock数据不一致？

**原因**：后端接口返回的数据结构与前端期望的不一致

**解决方法**：
1. 检查后端接口文档
2. 调整前端数据处理逻辑
3. 在Mock数据中模拟真实API的返回格式

## 注意事项

1. **开发阶段**：建议使用Mock模式，提高开发效率
2. **联调阶段**：切换到真实API模式，与后端联调
3. **测试阶段**：使用真实API模式，验证数据正确性
4. **上线前**：确保切换到真实API模式，并测试所有功能
5. **切换后**：重新编译小程序，验证数据是否正常显示
