## 1. 前置条件

### 1.1 导入新样式文件

在 `app.wxss` 顶部添加（放在所有现有样式之前）：

```css
/* 导入设计Token体系 */
@import "./styles/tokens.wxss";
@import "./styles/tailwind-like.wxss";
```

**说明**：
- `tokens.wxss` — 全局CSS变量定义（必须最先导入）
- `tailwind-like.wxss` — 工具类/组件预设（可选，也可直接在WXML中使用自定义属性）

### 1.2 备选：仅在首页导入

如果不想全局影响，可以在 `pages/index/index.wxss` 顶部导入：

```css
@import "../../styles/tokens.wxss";
@import "../../styles/tailwind-like.wxss";
```

> ⚠️ 建议全局导入，因为tokens是跨页面的设计体系。

## 7. backdrop-filter兼容方案

微信小程序的 `backdrop-filter`（毛玻璃效果）在以下环境可能存在兼容问题：

| 环境 | 兼容性 |
|------|-------|
| iOS 13+ / 微信 iOS 7.0.9+ | ✅ 完全支持 |
| Android 10+ / 微信 Android 7.0.12+ | ✅ 基本支持 |
| Android 低版本 / 低端机型 | ❌ 不支持 |

### 7.1 推荐方案：渐进增强

在 `tailwind-like.wxss` 中已提供降级类名：

```html
<!-- 方案A (推荐): 微信基础库2.10.0+ 使用 -->
<view class="glass-2">...</view>

<!-- 方案B (降级): 不支持backdrop-filter时 -->
<view class="glass-2-fallback">...</view>
```

### 7.2 JS检测降级（极致兼容）

在 `app.js` 或首页 `onLoad` 中添加检测：

```javascript
// 检测backdrop-filter支持
const supportsBackdrop = () => {
  return CSS.supports('backdrop-filter', 'blur(1px)') ||
         CSS.supports('-webkit-backdrop-filter', 'blur(1px)');
};

// 在Page的data中存储
this.setData({
  supportsBackdrop: supportsBackdrop()
});
```

在WXML中条件渲染：

```html
<view wx:if="{{supportsBackdrop}}" class="glass-2">
  <!-- 毛玻璃内容 -->
</view>
<view wx:else class="glass-2-fallback">
  <!-- 降级内容（同样内容，纯色背景） -->
</view>
```

### 7.3 各玻璃层的降级值速查

| 玻璃层 | 标准色 | 降级背景色 |
|-------|-------|----------|
| glass-1 | `rgba(255,255,255,0.04)` + blur(20px) | `rgba(15,15,19,0.88)` |
| glass-2 | `rgba(255,255,255,0.07)` + blur(30px) | `rgba(15,15,19,0.92)` |
| glass-3 | `rgba(139,92,246,0.12)` + blur(40px) | `rgba(30,20,50,0.92)` |
| nav-bar | `rgba(255,255,255,0.07)` + blur(30px) | `rgba(15,15,19,0.95)` |
| action-bar-comp | `rgba(255,255,255,0.07)` + blur(30px) | `rgba(15,15,19,0.92)` |

## 8. 注意事项

### 8.1 rpx与px混用
- 新Token体系全部使用 **rpx** 单位（微信小程序标准）
- 旧项目中混用的 `px` 值需要转为rpx（1px ≈ 2rpx在375设计稿下）
- 例外：`backdrop-filter: blur(Xpx)` 中的 `px` 保持px（blur不支持rpx）

### 8.2 文字对比度
- 暗色背景上，白色文字 `rgba(255,255,255,0.92)` 对比度约14:1 ✅
- `--text-tertiary` (`rgba(255,255,255,0.40)`) 在部分小字号下可读性略低，建议正文用 `--text-secondary`
- 在渐变背景上（如AI入口），使用 `--text-inverse` (`#0A0A18`)

### 8.3 Geist字体
- Geist 是Vercel开源字体，需在项目中引入
- 引入方式：将Geist字体文件放在 `miniapp/fonts/` 目录
- 在 `tokens.wxss` 中通过 `@font-face` 注册
- 如果字体加载失败，会自动降级到 `-apple-system, BlinkMacSystemFont, sans-serif`

```css
/* 在tokens.wxss 或 app.wxss 中添加 */
@font-face {
  font-family: 'Geist';
  src: url('/fonts/Geist-Regular.woff2') format('woff2');
  font-weight: 400;
  font-display: swap;
}
@font-face {
  font-family: 'Geist';
  src: url('/fonts/Geist-Medium.woff2') format('woff2');
  font-weight: 500;
  font-display: swap;
}
@font-face {
  font-family: 'Geist';
  src: url('/fonts/Geist-SemiBold.woff2') format('woff2');
  font-weight: 600;
  font-display: swap;
}
@font-face {
  font-family: 'Geist';
  src: url('/fonts/Geist-Bold.woff2') format('woff2');
  font-weight: 700;
  font-display: swap;
}
```

### 8.4 旧CSS变量回收
迁移完成后，请检查并移除以下旧CSS变量引用：
- `--primary`, `--primary-light`, `--primary-dark`
- `--text-hint`, `--text-muted`
- `--bg-card`, `--surface-1` ~ `--surface-4`
- `--border-color`
- `--shadow-sm`, `--shadow-md`
- `--spacing-xs` ~ `--spacing-xl`
- `--font-hint` ~ `--font-h3`
- 旧字体声明

### 8.5 过渡动效
- 原有的 `transition` 和 `animation` 推荐统一为弹簧曲线
- 按钮点击反馈: `transform: scale(0.98)` + `var(--ease-spring)`
- 卡片悬浮: `transform: scale(1.02)` + `var(--ease-spring)`
- 淡入淡出: `opacity` + `var(--ease-out)`
