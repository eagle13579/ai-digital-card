# 心智模型：异步SQLAlchemy + selectinload 铁律

**来源**：2026-07-05 AI数智名片 v3.4.1 复盘
**类型**：技术模式

## 核心

异步模式下，所有 `select(Model)` 查询必须加 `.options(selectinload(Model.relation))`，否则 Pydantic `model_validate` 时会触发 `MissingGreenlet` 错误。

## 适用范围

- 任何 async SQLAlchemy（`AsyncSession`）+ Pydantic `model_validate`
- 特别是 `Brochure.pages`、`User.profile` 等关系属性
- `db.refresh(obj)` 后访问关系属性同样会触发

## 标准写法

```python
from sqlalchemy.orm import selectinload

query = select(Model).options(selectinload(Model.relation))
result = await db.execute(query)
obj = result.scalars().first()
resp = ResponseModel.model_validate(obj)
```

# 心智模型：API版本重写中间件白名单

**来源**：2026-07-05 AI数智名片 v3.4.1 复盘
**类型**：架构模式

## 核心

`APIVersionRedirectMiddleware` 把 `/api/v1/xxx` 重写为 `/api/xxx`，导致注册在 `/api/v1/` 前缀的路由全部 404。所有 `/api/v1/*` 前缀必须加入白名单。

## 适用范围

- 任何使用 path 重写 middleware 的 FastAPI/Starlette 项目
- 路由注册在旧版本前缀但 middleware 自动重写

## 建议

- 自动收集所有已注册 route 的前缀，动态生成白名单
- 新增路由时必须更新白名单

# 心智模型：Python __pycache__ 部署陷阱

**来源**：2026-07-05 复盘
**类型**：运维模式

## 核心

`__pycache__/*.pyc` 缓存编译后的字节码，修改 `.py` 文件后 `git pull` 或文件修改不会自动使缓存失效。进程重启仍加载旧代码。

## 解决方案

```bash
find . -name "__pycache__" -type d -exec rm -rf {} +
find . -name "*.pyc" -delete
```

# 心智模型：微信小程序 token 生命周期管理

**来源**：2026-07-05 复盘
**类型**：前端模式

## 核心

- `wx.getStorageSync('token')` 可能读到过期 token
- 缓存 token 不能信任，需异步验证
- 每个 401 响应都应触发重新登录流程

## 标准流程

1. app onLaunch：读缓存 token → 调 `/api/v1/users/me` 验证
2. 返回 401 → `clearLogin()` → `reLaunch` 到登录页
3. request.js 全局 401 handler → 同样跳登录页
