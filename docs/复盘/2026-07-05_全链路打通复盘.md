## 三、关键决策

### 3.1 端口策略
- `:8001` — 僵尸进程（WSL命名空间无法从git-bash杀死），废弃
- **`:8002`** — `run.py` 启动，正式服务端口

### 3.2 API版本中间件
- `APIVersionRedirectMiddleware` 把 `/api/v1/xxx` 重写为 `/api/xxx`
- 导致大量 404（路由注册在 `/api/v1/...` 但被重写）
- 修复：34条前缀全部加入 `_EXPLICIT_V1_PREFIXES`

### 3.3 微信小程序登录
- 开发模式下 `__DEV_MODE__` 导致首页显示空白态不跳登录
- 移除该逻辑，无token一律跳登录页

### 3.4 定价同步
- 小程序硬编码定价与后端 `PRICING_CONFIG` 对齐
- Free¥0 / Pro¥99/月 / Enterprise¥499/月

## 四、心智模型提取

### 4.1 异步SQLAlchemy + selectinload
- **问题**：异步模式下懒加载关系属性导致 `MissingGreenlet`
- **模式**：所有 `select(Model)` 查询必须加 `.options(selectinload(Model.relation))`
- **适用范围**：任何用到 async SQLAlchemy + Pydantic `model_validate` 的场景
- **建议**：新建查询模板时默认加 `selectinload`

### 4.2 中间件白名单 vs API版本重写
- **问题**：Starlette middleware 按注册顺序处理，`APIVersionRedirectMiddleware` 重写路径后导致后面的 route 匹配失败
- **模式**：所有注册在 `/api/v1/` 前缀的路由必须在 middleware 白名单中声明
- **建议**：改为自动收集所有已注册 route 的前缀，动态生成白名单

### 4.3 Python `__pycache__` 陷阱
- **问题**：修改 `.py` 文件后进程重启但仍加载旧代码
- **模式**：`__pycache__/*.pyc` 缓存了编译后的字节码，`git pull` 或文件修改不会自动使缓存失效
- **建议**：每次部署/重启前执行 `find . -name "__pycache__" -type d -exec rm -rf {} +`

### 4.4 微信小程序 token 生命周期
- **问题**：`wx.getStorageSync('token')` 读到过期 token → app 认为已登录 → 所有 API 401
- **模式**：token 必须在 app 启动时异步验证，401 时自动清除并跳登录页
- **建议**：每个 401 响应都触发重新登录流程（不仅是清除 token）
