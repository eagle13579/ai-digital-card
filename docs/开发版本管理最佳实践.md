## 二、分支策略

```
master  ← 只接受PR合并，永远可部署
  └── dev-miniapp  ← 微信小程序开发分支，日常开发在此
       ├── feat/xxx  ← 新功能分支（可选）
       └── fix/xxx   ← 修复分支（可选）
```

**规则：**
- `master` — 基线分支。每次稳定版本合并到 master 后打 tag（如 `v1.0.0`）
- `dev-miniapp` — 日常开发分支。你每次说"开始开发"，我先切到这个分支
- `feat/xxx` / `fix/xxx` — 可选，大功能单独开分支，开发完合并回 dev-miniapp

## 三、标准开发流程（你只需要说"开发"，我自动执行）

```
你说"开发XX功能"
  ↓
Step 1: 定位  ← 我查注册表确认项目目录
  ↓
Step 2: 同步  ← git checkout dev-miniapp && git pull
  ↓
Step 3: 开发  ← 所有代码写入 D:\AI数智名片\miniapp\
  ↓
Step 4: 验证  ← 语法检查 + 微信开发者工具预览
  ↓
Step 5: 闭环节点（自动执行）
       git add -A
       git commit -m "feat: XX功能"
       git push origin dev-miniapp
  ↓
你验收 → 合并到 master + 打 tag
```

## 四、闭环节点（每次开发结束自动执行）

```bash
# 以下由白泽自动执行，你不需要管
git add -A
```

**不推送 = 不算完成。** 每次我说"做好了"之前，代码一定已推送到 GitHub。

## 七、常见问题

**Q: 开发到一半想切回来看旧版本？**
A: 我可以用 `git stash` 暂存当前改动，切回 master 看旧版，再切回来 `git stash pop` 恢复。

**Q: 7月3号那个19页版怎么找回？**
A: 在 `_archive/miniapp_current_20260704_193345/`，执行 `./scripts/restore-archive.sh miniapp_current_20260704_193345` 即可恢复。

**Q: 开发完成后怎么部署到阿里云？**
A: 代码推送到 master 后，登录阿里云 47.100.160.250 拉取最新代码部署。具体部署流程见 `deploy/` 目录。
