# 小程序修复→上传→手机测试 SOP v1.0

> 每次修改小程序代码后，严格按此流程操作。跳步=白做。

---

## Phase 1: 代码修复（本地）

### 1.1 开分支

```
git checkout develop
git pull origin develop
git checkout -b feature/<改动描述>
```

### 1.2 修代码

只改必要文件，遵循**最小改动原则**。

### 1.3 JS语法检查

微信小程序的 JS 语法比标准 JS 严格：

```bash
    catch(e){console.log(f+': ❌',e.message)}
  })"
```

### 1.4 提交

```bash
cd D:/AI数智名片
git add <改动的文件>
git commit -m "<改动类型>: <改动描述>"
```

## Phase 2: 上传到微信开发者工具

### 2.1 前提

微信开发者工具已登录并打开过该项目。

### 2.2 CLI上传

```bash
# 版本号递增规则: 修复小bug +0.0.1，加功能 +0.1.0
# 查看当前版本: 去微信公众平台版本管理看
# 上传命令:
"D:/微信web开发者工具/cli.bat" upload --project "D:/AI数智名片/miniapp" -v "1.0.9" -d "<改动描述>"
```

| 参数 | 说明 |
|:-----|:------|
| `v` 版本号 | 每次递增（1.0.5 → 1.0.6 → ... ） |
| `d` 描述 | 说明本次改了啥 |

### 2.3 验证上传成功

输出最后一行应为 `✔ upload`。如果报错：

| 报错 | 原因 | 解决 |
|:-----|:-----|:-----|
| `IDE server not running` | DevTools没启动 | 手动打开一次开发者工具 |
| `login first` | 未登录 | 手动打开DevTools扫码登录一次 |
| `invalid version` | 版本号重复 | 加大版本号 |

## Phase 3: 设体验版（手机测试用）

### 3.1 登录微信公众平台

浏览器打开 https://mp.weixin.qq.com

### 3.2 设置体验版

```
版本管理 → 往下翻 → 开发版本 → 找到刚上传的版本
→ 点「设为体验版」→ 确认
```

> ❗ 一定要设体验版，否则手机上打开的是「线上版本」（旧的）

### 3.3 添加体验成员（一次性操作）

```
版本管理 → 体验版 → 管理 → 添加微信号
（首次设置才需要，后续自动沿用）
```

## Phase 4: 手机测试

### 4.1 手机清理

```
手机 → 微信 → 「我」→「设置」→「通用」→「存储空间」
→ 找到「AI数智名片」→ 点「删除」（清理缓存）
→ 回到桌面 → 关掉微信后台重新打开
```

> ❗ 不清理缓存，手机可能跑的还是旧代码

### 4.2 扫码进入体验版

```
在微信公众平台 版本管理 页面
→ 体验版二维码 → 手机微信扫码
→ 或者让体验成员直接搜索小程序名称
```

### 4.3 测试清单

每个改动后必须测以下路径：

| 测试路径 | 预期结果 | 通过? |
|:---------|:---------|:-----:|
| 打开小程序 | 看到「登录页」（不是首页） | ☐ |
| 点「一键登录」 | 登录成功，跳首页 | ☐ |
| 首页加载 | 数据正常，无404/500报错 | ☐ |
| 创建画册 | 表单正常 → 生成成功 → 跳到预览页 | ☐ |
| 画册预览 | 4页可见（封面/资料/资源/联系） | ☐ |

### 4.4 看手机日志

如果出问题：

```
手机 → 微信 → 发现 → 小程序 → 右上角 ⋮
→ 打开调试 → 重新进入小程序
→ 底部出现 vConsole 按钮 → 点它看日志
→ 截图发给我
```

## 常见陷阱

| 场景 | 你以为 | 实际 |
|:-----|:-------|:-----|
| 修改了miniapp/代码 | 上传了就能用 | 还要设「体验版」手机才会更新 |
| 手机上看到旧效果 | 代码没修好 | 手机缓存没清 |
| DevTools编译后页面不变 | 代码没生效 | 点「编译」不是重置到首页，要手动清缓存 |
| 上传版本号用1.0.0 | 没问题 | 版本号不能重复，会被拒绝 |
