# 安全控制清单

## 1. 访问控制

### 1.1 基于角色的访问控制 (RBAC)
- **实现文件：** `app/middleware/rbac.py`, `app/models/rbac.py`
- **角色体系：** `admin`, `editor`, `viewer`, `api`
- **权限粒度：** 按资源类型 + 操作 (create/read/update/delete)
- **验证方式：** 每次请求通过中间件拦截验证

### 1.2 API Key 认证
- **实现文件：** `app/middleware/api_key.py`
- **密钥格式：** 32 字节随机字符串 + 前缀 `dc_`
- **存储方式：** bcrypt 哈希存储，原始值仅创建时返回
- **速率限制：** 每密钥每小时最多 1000 次请求

### 1.3 JWT 用户认证
- **实现文件：** `app/routers/auth.py`
- **Token 类型：** Bearer JWT (HS256)
- **过期时间：** 24 小时（Access Token）
- **Refresh Token：** 支持静默续期

## 2. 加密

### 2.1 传输加密 (TLS)
- **协议：** TLS 1.2 / 1.3
- **配置：** 反向代理层 (Nginx / Cloudflare)
- **HSTS：** `max-age=31536000; includeSubDomains`
- **证书管理：** Let's Encrypt / 自动化续期

### 2.2 静态加密
- **数据库：** 全量磁盘级加密 (LUKS / AWS RDS 加密)
- **备份文件：** AES-256-GCM 加密存储
- **敏感字段：** 密码使用 bcrypt 哈希存储
- **密钥管理：** 环境变量注入，生产环境通过密钥管理服务 (KMS)

## 3. 监控与可观测性

### 3.1 OpenTelemetry 分布式追踪
- **实现文件：** `app/middleware/otel.py`
- **采样策略：** 生产环境 10% 采样率，错误链路 100%
- **导出目标：** OTLP 兼容后端 (如 Grafana Tempo)

### 3.2 Grafana + SLO 监控
- **实现文件：** `app/slo_tracker.py`
- **监控指标：**
  - API 响应时间 p50 < 200ms, p95 < 1s
  - 错误率 < 0.1%
  - 可用性 >= 99.9%
- **仪表板：** Grafana 面板实时展示 SLO 达成率

### 3.3 告警机制
- **实现文件：** `app/database_monitor.py`
- **告警规则：**
  - 数据库连接池利用率 > 80%
  - 慢查询 > 2s
  - 错误率突增 > 200%
- **通知渠道：** 企业微信 / 邮件 / PagerDuty

## 4. 备份与灾难恢复

### 4.1 数据库备份
- **脚本：** `scripts/backup.sh`
- **备份频率：** 每日全量 + 每 6 小时增量
- **保留策略：** 7 天本地 + 30 天远程 (S3)
- **恢复演练：** 每季度一次

### 4.2 灾难恢复
- **RPO (恢复点目标)：** ≤ 6 小时
- **RTO (恢复时间目标)：** ≤ 2 小时
- **多 AZ 部署：** 生产环境跨可用区冗余

## 5. 事件响应计划

### 5.1 事件分类

| 级别 | 定义 | 响应时间 | 通知对象 |
|------|------|----------|----------|
| P0 | 数据泄露 / 服务完全不可用 | ≤ 15 分钟 | CTO + 安全团队 |
| P1 | 核心功能受损 / 用户数据受影响 | ≤ 30 分钟 | 技术负责人 |
| P2 | 非核心功能异常 | ≤ 4 小时 | 开发团队 |
| P3 | 低影响问题 | ≤ 24 小时 | 迭代计划 |

### 5.2 响应流程
1. **检测：** 监控告警 / 用户报告
2. **评估：** 判断影响范围与严重级别
3. **遏制：** 立即执行预定义遏制措施
4. **根因分析：** 定位问题根源
5. **修复：** 部署修复方案
6. **复盘：** 48 小时内完成事后分析报告

### 5.3 演练计划
- 每季度一次桌面推演
- 每年一次全生产环境模拟演练
