## 模式一：Gateway 接入层

链客宝统一网关 `:5136` 负责所有外部请求的统一接入。

| 组件 | 说明 |
|:-----|:------|
| **统一网关** | 链客宝网关 :5136，提供统一API入口 |
| **CORS中间件** | FastAPI CORS，全来源允许（开发阶段） |
| **JWT认证** | Bearer Token，注册/登录/微信Mock |
| **路由注册** | 18个API端点注册在 FastAPI app 上 |

```
客户端(H5/微信/扫码) → 链客宝网关(:5136) → AI数字名片(:8003)
```

## 模式三：delegate 委托执行层

匹配引擎、画册渲染、信任计算等子任务以委派方式执行，可并行或串行。

### 3.1 匹配引擎

```
输入: source_user + 全局画册数据
  │
  ├─ 余弦相似度计算 (40%权重)
  │   ├─ provide_tags × need_tags → 点积
  │   ├─ 向量归一化
  │   └─ manual_source ×3.0 加权
  │
  ├─ 标签供需重叠计算 (60%权重)
  │   ├─ A.provide ∩ B.need → common_tags
  │   └─ 加权评分
  │
  └─ 结果排序 (降序) → 返回 Top-N 匹配
```

### 3.2 信任网络计算

```
直接信任 ─→ 间接信任 ─→ 协同信任 ─→ 社区信任
  (1度)      (2度)       (共同标签)    (全网)
```

### 3.3 画册渲染

- 内存缓存 BROCHURES dict 作为主数据源
- SQLite 持久化确保数据不丢失
- 写入时双写: 内存 + SQLite
- 读取时优先内存，缺失则回退 SQLite

### 3.4 OCR扫描管线 (预留)

- pdfplumber 文本提取
- PIL 图像预处理
- 字段 CN Regex 识别
- DeepSeek AI摘要提取

子任务委派模式：
```
委派请求 → 引擎层 → AI提取器/OCR扫描器/匹配引擎
              ↓
         返回结果 → 更新画册数据 → 同步到 SQLite
```

## 链客宝生态融合 (Phase 3 · v2.2.0)

### 概述

AI数字名片 v2.2 与链客宝企盟匹配系统深度打通，实现信任网络数据跨平台同步与供需匹配推荐。

### 架构变更

```
AI数字名片 (:8003)               链客宝网关 (:5136)
        │                              │
        ├── 信任网络数据 ──────────────→  POST /api/external/brochure/trust-sync
        │                              │
        └── 推荐匹配结果 ←──────────────  GET  /api/external/brochure/matches
```

### 新增网关路由 (gateway.py)

| 路由前缀 | 目标后端 | 说明 |
|:---------|:---------|:-----|
| `/api/brochure/*` | `brochure` (:8003) | AI数字名片主路由 |
| `/api/auth/*` | `brochure` (:8003) | 注册/登录/me/登出 |
| `/api/tag/*` | `brochure` (:8003) | 标签管理 |
| `/api/match/*` | `brochure` (:8003) | 匹配引擎 |
| `/api/external/*` | `brochure` (:8003) | 外部对接接口 |

### 桥接接口 (chainke_bridge.py)

新增两个核心桥接函数，依赖 Python 标准库 `urllib`，零额外依赖：

| 函数 | 方向 | 描述 |
|:-----|:-----|:------|
| `sync_trust_to_chainke(user_id, trust_data)` | 推送 | 将信任网络数据推送到链客宝 |
| `sync_matches_from_chainke(user_id)` | 拉取 | 从链客宝拉取推荐匹配 |

**信任数据格式**（TrustNetwork 模型映射）：
```
trust_data = {
    "trust_value": {
        "core_competence": [...],      # 核心能力（来自 provide 标签）
        "industry_experience": [...],  # 行业经验
        "cooperation_advantage": [...] # 合作优势
    },
    "resource_needs": {
        "resources_required": [...],    # 资源需求（来自 need 标签）
        "ideal_partner_profile": [...]  # 理想合作伙伴画像
    },
    "trust_evidence": {
        "cooperation_cases": [...],     # 合作案例
        "client_evaluations": [...]     # 客户评价
    },
    "trusted_user_ids": [...]           # 信任关系用户 ID 列表
}
```

### 新增 API 端点

| 端点 | 方法 | 认证 | 描述 |
|:-----|:-----|:-----|:------|
| `/api/brochure/sync/chainke` | GET | 需登录 | 手动触发与链客宝的数据同步 |

**响应格式**：
```json
{
    "code": 0,
    "message": "同步完成",
    "data": {
        "trust_pushed": 1,
        "matches_pulled": 5,
        "source": "chainke",
        "matches": [...]
    }
}
```

### 故障降级策略

链客宝后端不可达时自动 fallback 到本地匹配引擎：
1. 尝试导入 `digital_brochure_api.py` 中的 `run_matching()` 匹配引擎
2. 若导入失败，返回空匹配列表
3. 所有降级行为记录日志，不抛出异常
