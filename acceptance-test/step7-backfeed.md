# Step 7 · 反哺

## 心智模型提取

从本次20人验收测试中提取3个可复用心智模型：

### 心智模型1: Auth先验检查法则
在验收测试前，先检查认证系统的可用性(auth/register + auth/login)。如果认证不可用，立即调整策略——90%的需认证API会阻塞，剩余的测试精力应转向公开API+源码审计+架构审计。

### 心智模型2: 验收测试的收敛法则
20人验收不需要真的派20人。3批×3人=9人即可覆盖所有维度。关键不是人数，是角色多样性(CEO/跨境/安全/合规/架构/UI/路由)。

### 心智模型3: 融合后的系统性冒烟测试
跨项目融合(链客宝→AI数智名片)后，必须做系统性冒烟测试：
1. ORM模型冲突检查(同名类/同名tablename)
2. 中间件兼容性检查(CSRF白名单是否包含新endpoint)
3. 认证一致性检查(两套认证是否冲突)
4. 路由注册完整性(新路由是否都已include到app)
