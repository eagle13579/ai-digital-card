## 🚨 P0 全球阻塞问题 (3个)

### P0-A. ❌ ORM Deal类名冲突 — 影响18+端点500

| 属性 | 内容 |
|:-----|:------|
| **根因** | 链客宝融合后，`Deal` 类在 `escrow.py`(→escrow_deals) 和 `workflow/models/deal.py`(→deals) 双重映射 → SQLAlchemy声明式基类冲突 |
| **症状** | 注册/登录时第一次触发懒加载 → 500。所有CRM/知识图谱/六度人脉/心智模型/画布端点全部500 |
| **影响** | 认证系统崩溃(用户无法注册/登录)、CRM全模块(18+端点)、心智模型(3端点)、知识图谱(2端点)、六度人脉(3端点) |
| **修复** | 重命名其中一个Deal类，或使用 `__tablename__` 区分 |

### P0-B. ❌ CSRF防护过度 — 阻塞所有POST公共端点

| 属性 | 内容 |
|:-----|:------|
| **根因** | CSRF中间件拦截了所有POST请求，包括标注为"公开"的端点(bot webhook/crm form/learning trigger) |
| **症状** | Slack/飞书/钉钉回调全部403、CRM嵌入式表单不可用、AI学习触发器不可用、设计评审API不可用 |
| **影响** | 所有POST公共端点(charity-callable/embed-callable)全部失效 |
| **修复** | CSRF中间件增加白名单路由，或对公开端点跳过CSRF校验 |

### P0-C. ❌ GDPR合规零基础 — 评分6/100

| 属性 | 内容 |
|:-----|:------|
| **缺失** | 隐私政策页面、服务条款页面、GDPR路由未注册、数据保留策略、用户同意记录、数据可移植性端点 |
| **状态** | GDPR路由代码已写(`app/routers/gdpr.py`)但只差一行`app.include_router()` |
| **影响** | 面向欧盟用户的部署不可行，违反Art.5/6/7/12/13/15/17/20/30/37 |
