## 1. 产品定位

### 一句话定位

> **链客宝AI数字名片是面向企业家的智能供需匹配平台 — 一分钟创建3D翻页电子画册，AI标签匹配自动推荐合作机会，付费会员解锁真实联系方式，实现高效商业连接。**

### 三个核心价值

| # | 价值 | 说明 |
|:-:|:-----|:------|
| 1 | **秒级建册** | 填写基本信息 + 双维度标签（我能提供/我需要），AI自动生成3D翻页电子画册，替代传统纸质名片 |
| 2 | **智能匹配** | 基于供需标签重叠度 + 语义相似度 + 权重加分的三层匹配引擎，精准推荐合作对象 |
| 3 | **付费社交** | 匹配结果脱敏展示（不显示联系方式），付费会员才能解锁查看真实联系人，建立高质量商业连接壁垒 |

### 与链客宝生态的关系

```
链客宝主站（会员体系+支付+用户主库）
    ↕ 链客宝桥接（已有 chainke_bridge.py）
AI数字名片服务（画册+匹配+标签+信任网络）
    ↕ API /api/v1/*
微信小程序（前端展示+填单+分享+浏览）
```

**核心原则**：名片身份 = 链客宝会员身份，打通。不同会员等级解锁不同名片权益。

## 2. 用户画像

### 三类核心用户

#### 👤 类型A：企业主/创业者（核心用户）
| 属性 | 特征 |
|:-----|:------|
| 身份 | 中小企业老板、创业者、自由职业者 |
| 痛点 | 线下社交效率低，名片纸质化难管理，找不到精准合作对象 |
| 需求 | 快速创建专业电子名片，被有需求的合作伙伴发现，获取高质量商机 |
| 会员倾向 | Gold（¥999/年）或 Diamond（¥4,999/年） |
| 日均使用 | 每天浏览3-5次匹配推荐，每周主动查看10+名片 |

#### 👤 类型B：商务/销售负责人
| 属性 | 特征 |
|:-----|:------|
| 身份 | 市场总监、大客户销售、商务拓展 |
| 痛点 | 需要大量交换名片，但后续跟进困难，缺乏精准客户筛选工具 |
| 需求 | 通过匹配引擎找到潜在客户，查看对方供需标签后精准触达 |
| 会员倾向 | Gold（¥999/年） |
| 日均使用 | 每天查看匹配结果20+次，频繁使用解锁功能 |

#### 👤 类型C：平台访客/受邀浏览者
| 属性 | 特征 |
|:-----|:------|
| 身份 | 被分享名片链接吸引而来的微信用户（可能未注册） |
| 痛点 | 初次接触平台，需要低门槛了解服务价值 |
| 需求 | 浏览他人电子画册、感受翻页效果和匹配概念 |
| 会员倾向 | Free（免费）→ 被匹配结果吸引后升级 |
| 行为 | 通过微信分享/二维码进入，浏览画册后注册成为正式用户 |

## 3. 核心链路

### 六步完整链路

```
① 发出名片
   │
   ├─ 用户在小程序创建/编辑电子画册
   ├─ 填写个人信息 + "我能提供"标签 + "我需要"标签
   └─ 生成分享链接/二维码
   
② 浏览名片
   │
   ├─ 对方通过微信分享/扫码打开电子画册
   ├─ 3D翻页效果展示（StPageFlip）
   ├─ 展示个人信息、公司、供需标签
   └─ 无需登录即可浏览（公开访问）
   
③ 填写信息/表达兴趣
   │
   ├─ 访客在浏览页面表达兴趣
   ├─ 填写姓名+联系方式（可选留资）
   └─ 触发访客记录（visitor_logs）
   
④ 入库
   │
   ├─ 访客信息存入数据库
   ├─ 若访客是已注册用户，自动建立用户关联
   └─ 若访客是新用户，触发注册引导
   
⑤ 匹配
   │
   ├─ 匹配引擎运行（三层评分）
   │    ├─ 标签重叠度（40%）：provide×need 匹配
   │    ├─ 语义相似度（40%）：NLP向量匹配
   │    └─ 权重加分（20%）：信任网络+等级权重
   ├─ 匹配结果持久化到 match_records
   └─ 匹配结果脱敏展示（联系方式隐藏）
   
⑥ 传播
   │
   ├─ 用户分享画册到微信好友/群/朋友圈
   ├─ 每个分享带 share_token 追踪
   ├─ 传播链记录（谁→谁→谁）
   └─ 新访客进入 → 回到步骤②，形成裂变
```

## 5. 页面流程

### 5.1 小程序页面架构

```
app.json 注册页面（共12页）
│
├── 底部Tab（3页）:
│   ├── pages/index/           首页（名片展示+推荐匹配）
│   ├── pages/pool/            匹配池（匹配结果列表）
│   └── pages/mine/            我的（个人中心+管理入口）
│
├── 画册主流程（4页）:
│   ├── pages/brochure/index   电子画册预览页（3D翻页H5）
│   ├── pages/brochure-form/   填单页（信息填写+标签）
│   ├── pages/brochure-qrcode/ 二维码名片页
│   └── pages/webview/         通用WebView（H5翻页备用）
│
├── 匹配与社交（3页）:
│   ├── pages/match-result/    匹配结果页（脱敏展示+付费解锁）
│   ├── pages/trust-network/   信任网络管理页
│   └── pages/visitor-log/     访客记录页
│
├── 会员与数据（2页）:
│   ├── pages/membership/      会员权益页
│   └── pages/data-board/      数据看板页
│
└── 基础页（复用现有）:
    ├── pages/login/           登录
    ├── pages/register/        注册
    └── pages/notifications/   通知
```

### 5.2 页面详情

---

#### 页面1：首页 — 名片展示（`pages/index/index`）

**用途**：用户名片主页 + 画册展示 + 快速入口

**功能点**：
1. **顶部个人信息卡片**
   - 头像、姓名、职位、公司
   - 会员等级徽章（Free/Gold/Diamond/Board 不同颜色）
   - 今日匹配推荐数角标
2. **电子画册预览入口**
   - 显示画册封面缩略图
   - 点击跳转至 `brochure/index` 全屏翻页模式
   - 显示「总浏览量」「访客数」概览
3. **快速操作栏**
   - 「编辑画册」→ brochure-form
   - 「我的二维码」→ brochure-qrcode
   - 「分享名片」→ 唤起微信分享组件
4. **智能匹配推荐区**（3个卡片）
   - 显示匹配分数最高的3个潜在合作对象
   - 卡片展示：对方头像、公司、匹配度百分比、供需标签交集
   - 点击 → 跳转匹配结果详情
   - **联系方式不显示，显示「待解锁」按钮**
5. **底部Tab栏**
   - 首页 | 匹配池 | 我的

**状态**：
| 状态 | 显示 |
|:-----|:------|
| 无画册 | 引导创建 button：「创建你的第一本电子画册」 |
| 已创建 | 展示画册封面+数据 |
| 加载中 | 骨架屏（头像+标题占位） |
| 网络失败 | 错误页+重试按钮 |

---

#### 页面2：填单页（`pages/brochure-form/index`）

**用途**：创建/编辑电子画册，填写个人信息+供需标签

**功能点**：
1. **分步表单（4步向导）**
   - **Step 1 — 基础信息**
     - 头像上传（裁剪+压缩）
     - 姓名、职位、公司名称、手机号
     - 个人简介（200字以内）
   - **Step 2 — 我能提供（provide 标签）**
     - 搜索/选择标准标签库
     - 支持自定义输入（Gold以上）
     - 展示已选标签，可删除
     - 免费版上限3个，Gold以上不限
   - **Step 3 — 我需要（need 标签）**
     - 同上布局
     - 明确标示：「请告诉我你正在寻找什么合作伙伴/资源」
   - **Step 4 — 预览+发布**
     - 3D画册效果预览（嵌入StPageFlip）
     - 确认无误后点击「发布」
2. **标签智能推荐**
   - 输入公司/职位后，AI推荐相关标签
   - 推荐来源：已有用户标签库 + 行业热词
3. **保存草稿**
   - 每一步自动保存草稿到本地缓存
   - 断网续填
4. **图片处理**
   - 头像自动压缩至200KB以内
   - WebP格式输出

**交互**：
- 底部进度条指示（1/4, 2/4, 3/4, 4/4）
- 上一步/下一步按钮
- 标签选中动画（填充颜色变化）

---

#### 页面3：电子画册预览页（`pages/brochure/index`）

**用途**：3D翻页电子画册全屏浏览

**功能点**：
1. **3D翻页引擎**
   - 基于StPageFlip的3D翻页效果
   - 模拟真实纸质感（阴影+翻页弧度）
   - 支持手指左右滑动翻页
   - 缩略图导航（底部小点指示）
2. **画册内容页**
   - **封面页**：姓名、职位、公司、会员徽章、头像
   - **关于我页**：个人简介 + 联系方式（脱敏，仅付费可见）
   - **我能提供页**：provide标签列表（每个标签带图标）
   - **我需要页**：need标签列表（带紧急程度标记）
   - **信任背书页**：信任关系列表（仅显示人数和行业）
   - **联系我页**：仅Gold以上显示「解锁联系方式」按钮
3. **底部操作栏**
   - 「✨ 匹配TA」→ 运行匹配引擎，跳转匹配结果
   - 「📤 分享」→ 微信分享（带share_token）
   - 「📱 保存二维码」→ 保存名片二维码到相册
4. **访客记录触发**
   - 打开画册即记录访客日志
   - 记录：openid（如有）、浏览时间、浏览页数

**特殊模式**：
| 模式 | 触发条件 | 效果 |
|:-----|:---------|:------|
| 本人浏览 | 当前用户=画册主人 | 显示编辑按钮，不记录访客 |
| 他人浏览 | 其他注册用户 | 记录访客，显示匹配按钮 |
| 匿名浏览 | 未登录访客（通过分享链接） | 只记录IP，底部显示「注册后查看更多」 |

---

#### 页面4：匹配结果页（`pages/match-result/index`）

**用途**：展示AI匹配结果，脱敏显示，付费解锁联系方式

**功能点**：
1. **匹配概览**
   - 匹配分数（百分比显示，带颜色：>80%绿，>60%黄，<60%灰）
   - 匹配维度分解：供需匹配 X%，行业相关 X%，信任背书 X%
   - 共同标签列表：「你们都关注了XXX」
2. **对方信息（脱敏）**
   - 头像（模糊处理/小图）
   - 行业 + 职位（公开）
   - 公司名称（公开）
   - **手机号/微信：显示「*** **** ****」**
   - **微信号：显示「***」**
3. **供需匹配详情**
   - 「对方能提供」匹配了你的哪些需求
   - 「对方需要」匹配了你的哪些供给
   - 高亮显示匹配上的标签
4. **解锁联系方式按钮**
   - **按钮文案**：「对接券解锁 ¥X」（X = 当前会员折扣价）
   - 点击 → 检查对接券余额 → 有券 → 直接解锁 → 显示完整联系方式
   - 无券 → 跳转购买对接券/升级会员
   - 至尊会员（Board）：免费解锁，无次数限制
5. **动作按钮**
   - 「发送好友申请」→ 触发信任网络请求
   - 「收藏此匹配」→ 保存到收藏夹
   - 「不感兴趣」→ 反馈降低此类匹配权重

**关键规则**：
- ⚠️ **所有非付费会员的匹配结果均不展示联系方式**
- 匹配结果默认显示脱敏数据
- 解锁后24小时内可反复查看，不重复扣券
- 解锁记录存入 match_records.contact_unlocked

---

#### 页面5：我的名片管理页（`pages/mine/index`）

**用途**：个人信息+画册管理+会员中心

**功能点**：
1. **用户信息头部**

### 7.2 解锁后规则

| 规则 | 说明 |


```
    │
    ├─ 主动触发: 用户点击「✨ 匹配TA」按钮
    │   POST /api/v1/match
    │   Body: { "source_user_id": 123, "target_user_id": 456 }
    │
    └─ 被动触发: 定时任务批量运行匹配引擎

    │
    ▼
匹配引擎执行
    ├─ 1. 标签重叠度 (40%)
    │   ├─ source.provide_tags × target.need_tags → 点积
    │   ├─ source.need_tags × target.provide_tags → 点积
    │   └─ 双向匹配总分
    │
    ├─ 2. 语义相似度 (40%)
    │   ├─ 文本向量化 (DeepSeek/M3E)
    │   ├─ 余弦相似度计算
    │   └─ 阈值过滤 (<0.3 跳过)
    │
    ├─ 3. 权重加分 (20%)
    │   ├─ 信任网络同圈 +10
    │   ├─ 同行业 +5
    │   ├─ 高会员等级权重 +5~15
    │   └─ manual_source标记 ×3.0
    │
    └─ 结果: 加权总分 (0~100)
    │
    ▼
结果持久化
    ├─ 写入 match_records 表
    │   user_a / user_b / score / common_tags / status / created_at
    │
    ├─ 脱敏处理: 联系方式字段不返回
    │
    └─ 存入Redis缓存（热门匹配加速）
    │
    ▼
通知推送
    ├─ 匹配分数 > 70% → 即时推送（微信订阅消息）
    ├─ 匹配分数 50-70% → 每日汇总推送
    └─ 匹配分数 < 50% → 不推送，仅在匹配池展示
    │
    ▼
用户查看
    ├─ 打开 match-result 页 → GET /api/v1/match/{id}
    ├─ 展示脱敏信息
    └─ 「解锁联系方式」→ 走付费解锁流程
```
