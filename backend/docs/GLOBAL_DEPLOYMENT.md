## 2. 区域划分与基础设施

### 2.1 三区域架构

| 区域 | 覆盖范围 | 云服务商 | K8s 集群 | 区域DNS |
|------|---------|---------|---------|---------|
| **中国区 (CN)** | 中国大陆 | 阿里云 (杭州/上海) | ACK | cn.liankebao.top |
| **亚太区 (APAC)** | 东南亚、日韩、澳洲 | AWS Singapore (ap-southeast-1) | EKS | ap.liankebao.top |
| **美欧区 (AMER)** | 北美、南美、欧洲 | AWS US West (us-west-2) | EKS | us.liankebao.top |

### 2.2 中国区特殊说明

- **备案要求**: 中国大陆节点必须完成 ICP 备案
- **合规**: 数据本地化存储，用户数据不出境
- **服务商**: 阿里云为主要从服务商，备选华为云/腾讯云
- **加速**: 阿里云 DCDN (全站加速) + 动态加速
- **域名**: liankebao.cn (中国), liankebao.top (国际)

### 2.3 全球加速方案

| 方案 | 适用场景 | 备注 |
|------|---------|------|
| **AWS Global Accelerator** | 动态API加速 | Anycast IP，跨区域智能路由 |
| **阿里云全球加速 GA** | 中国区动态加速 | 跨境合规加速 |
| **Cloudflare Argo Smart Routing** | 非中国区优化 | 智能源站路由 |

## 3. CDN 策略

### 3.1 静态资源 CDN

| 资源路径 | 缓存策略 | CDN TTL | 说明 |
|---------|---------|--------|------|
| `/assets/*` | `public, immutable` | 1年 | Vite hash 构建产物 |
| `*.js` / `*.css` | `public, immutable` | 1年 | hash 文件名 |
| `*.png/jpg/svg/ico` | `public, immutable` | 1年 | 图片/图标 |
| `*.woff2/ttf` | `public, immutable` | 1年 | 字体文件 |
| `/index.html` | `no-cache` | 0 | SPA入口，必须回源 |
| `/manifest.json` | `public, must-revalidate` | 7天 | PWA清单 |
| `/sw.js` | `no-cache` | 0 | Service Worker |

### 3.2 动态内容加速

| 路径 | 缓存策略 | 说明 |
|------|---------|------|
| `/prerendered/*` | `public, max-age=1800` | 预渲染页面，30分钟缓存 |
| `/api/public/*` | `public, max-age=60` | 公共API，1分钟缓存 |
| `/api/*` | `no-cache` | 动态API，不缓存 |
| `/health` | `no-cache` | 健康检查 |

### 3.3 边缘计算 (CDN Edge Workers)

Cloudflare Workers / Lambda@Edge 执行:

1. **语言检测与路由** — 根据 `Accept-Language` 或 GeoIP 自动重定向
2. **爬虫识别与SSR** — 识别搜索引擎爬虫 → 返回预渲染 HTML
3. **Geo 路由** — 根据用户IP调度到最近区域
4. **A/B 测试** — 边缘层分流
5. **请求聚合/修改** — 添加安全头、修改路径

## 4. 边缘计算详细设计

### 4.1 Cloudflare Workers 功能

```javascript
// 核心工作流 (详见 deploy/cdn/cloudflare-workers.js)
//
// 1. 请求进入 Workers
// 2. 爬虫检测 → 是 → 返回静态预渲染 HTML (从 CDN 缓存)
// 3. 语言检测 → 非中文 → 302 重定向到对应语言路径
// 4. 静态资源 → 直接返回缓存
// 5. API 请求 → 转发到 Global Accelerator → 最近区域后端
// 6. SSR 渲染 → 在 Edge 执行轻量渲染或代理到区域 prerender 服务
```

### 4.2 SSR 在 Edge 的执行策略

```
┌──────────────────────────────────────────────────────┐
│                  爬虫请求流程图                        │
│                                                      │
│  爬虫请求 → CDN Edge Workers                         │
│    ├─ 检查: 静态预渲染存在?                          │
│    │   ├─ 是 → 返回预渲染 HTML (CDN缓存命中)         │
│    │   └─ 否 → 请求区域 Prerender 服务               │
│    │       ├─ 成功 → 返回渲染 HTML + 写入缓存        │
│    │       └─ 失败 → 返回 SPA index.html (降级)     │
│    └─ 记录: 哪些页面需要预渲染 (统计分析)            │
└──────────────────────────────────────────────────────┘
```

## 5. Kubernetes 全球集群设计

### 5.1 每个区域的 K8s 集群配置

```
区域集群组件:
├── Namespace: chainke-{region}    # 区域化命名空间
├── Deployment:
│   ├── web (Next.js SPA)          # 2〜20 pod (HPA)
│   ├── api (FastAPI)              # 2〜10 pod (HPA)
│   └── prerender (Puppeteer SSR)  # 1〜5 pod (HPA)
├── Service:
│   ├── web-service (LoadBalancer) # 区域负载均衡
│   └── api-service (ClusterIP)    # 内部服务发现
├── ConfigMap:
│   ├── region-config              # 区域配置
│   └── feature-flags              # 特性开关
├── HPA (CPU > 70% → 扩容)
└── PodDisruptionBudget (min 2 pods)
```

### 5.2 跨区域服务发现

```yaml
# 方案: Istio Service Mesh + 跨区域ServiceEntry
# 区域间通过 AWS PrivateLink / 阿里云私网连接通信
#
# China → APAC: 阿里云 CEN + AWS Direct Connect 或公网 TLS
# APAC → AMER: AWS PrivateLink
# China → AMER: 通过 APAC 中转或直接公网加密
```

### 5.3 自动扩缩容策略

| 指标 | 阈值 | 扩容策略 | 说明 |
|------|------|---------|------|
| CPU 使用率 | >70% | 按比例扩容 | 主要指标 |
| 内存使用率 | >80% | 按比例扩容 | 辅助指标 |
| 请求 QPS | >1000/实例 | 按比例扩容 | 自定义指标 |
| P99 延迟 | >500ms | 按比例扩容 | 自定义指标 |

## 9. 安全合规

### 9.1 区域合规要求

| 区域 | 要求 | 措施 |
|------|------|------|
| 中国 | 等保2.0 / 数据本地化 | 阿里云政务/金融云合规区 |
| 欧洲 | GDPR | 数据分类 + 用户删除API |
| 美国 | SOC2 / CCPA | 审计日志 + 数据最小化 |
| 全球 | SSL/TLS 1.2+ | Let's Encrypt / 商业证书 |

### 9.2 安全策略

- **WAF**: 每个区域启用区域 WAF (阿里云 WAF / AWS WAF)
- **DDoS**: CDN 层 DDoS 防护 (Cloudflare / AWS Shield)
- **mTLS**: 服务间通信使用 mTLS (Istio)
- **证书**: 每个区域独立 TLS 证书，自动续期 (cert-manager)
