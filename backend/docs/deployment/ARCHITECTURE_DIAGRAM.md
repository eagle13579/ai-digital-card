## 2. 数据流图

```
  ┌──────────┐     ┌──────────────┐     ┌────────────┐     ┌──────────┐     ┌──────────┐
  │  前端 SPA  │────▶│  API Gateway  │────▶│  Router 层  │────▶│ Service 层│────▶│  Model 层 │
  │ (React)   │     │  (/api/*)     │     │  (路由分发)  │     │  (业务逻辑) │     │  (ORM/DB) │
  └──────────┘     └──────────────┘     └────────────┘     └──────────┘     └─────┬────┘
       │                  │                  │                  │                  │
       │ 请求/响应          │ 代理转发           │ 路径匹配           │ 业务处理           │
       ▼                  ▼                  ▼                  ▼                  ▼
  ┌──────────┐     ┌──────────────┐     ┌────────────┐     ┌──────────┐     ┌──────────┐
  │ HTTPS    │     │ Nginx proxy   │     │ FastAPI    │     │ Service  │     │ SQLAlchemy│
  │ 443      │     │ pass + 安全头  │     │ APIRouter  │     │ 逻辑      │     │ + asyncpg │
  └──────────┘     └──────────────┘     └────────────┘     └──────────┘     └──────────┘
                                                                                │
                                                                                ▼
                                                                          ┌──────────┐
                                                                          │PostgreSQL │
                                                                          │  16       │
                                                                          └──────────┘

  详细请求处理流水线:

   用户请求
     │
     ├─▶ DNS 解析 (liankebao.top → 47.116.116.87)
     │
     ├─▶ Nginx 443 端口
     │     ├─ SSL 握手 + TLS 1.2/1.3
     │     ├─ 安全头注入 (CSP, HSTS, X-Frame-Options)
     │     ├─ 爬虫检测 (User-Agent map)
     │     │    ├─ 爬虫 → 预渲染 HTML 优先 (prerendered/*)
     │     │    │         ├─ 静态文件存在 → 直接返回
     │     │    │         └─ 静态文件不存在 → @prerender-dynamic (Puppeteer)
     │     │    │                              └─ 不可用 → @spa-fallback
     │     │    └─ 普通用户 → SPA 正常流程
     │     ├─ Accept-Language 检测 → 非中文浏览器自动重定向 /ko/ /en/
     │     ├─ 多语言前缀剥离 (/en/xxx → /xxx)
     │     ├─ 静态资源 → 直接返回 (缓存控制)
     │     └─ API 请求 (/api/*) → 反向代理到 FastAPI
     │
     ├─▶ FastAPI 中间件链 (7层)
     │     ① CORSMiddleware     — 跨域允许
     │     ② RequestLogMiddleware — 请求日志 (method, path, status, duration)
     │     ③ RateLimitMiddleware  — 基于 Redis 的速率限制
     │     ④ AuthMiddleware       — JWT Token 解析与验证
     │     ⑤ RoleMiddleware       — 角色权限检查
     │     ⑥ AuditMiddleware      — 关键操作审计日志
     │     ⑦ ExceptionMiddleware  — 全局异常捕获与统一格式化
     │
     ├─▶ Router 层
     │     ├─ /api/auth/*      — 认证注册登录
     │     ├─ /api/users/*     — 用户管理
     │     ├─ /api/products/*  — 产品/商品
     │     ├─ /api/orders/*    — 订单
     │     ├─ /api/payment/*   — 支付 (含回调)
     │     ├─ /api/contacts/*  — 人脉
     │     ├─ /api/needs/*     — 需求
     │     ├─ /api/activities/*— 活动
     │     ├─ /api/promoter/*  — 推广大使
     │     └─ /health          — 健康检查
     │
     ├─▶ Service 层 (业务逻辑)
     │     ├─ AuthService        — 认证/授权
     │     ├─ UserService        — 用户 CRUD
     │     ├─ ProductService     — 产品管理
     │     ├─ OrderService       — 订单流程
     │     ├─ PaymentService     — 支付网关集成
     │     ├─ ContactService     — 人脉匹配
     │     ├─ NeedService        — 需求匹配
     │     ├─ ActivityService    — 活动管理
     │     └─ PromoterService    — 推广大使结算
     │
     └─▶ Model 层 (SQLAlchemy ORM)
           ├─ User, Product, Order, Payment
           ├─ Contact, Need, Activity
           └─ Promoter, Withdrawal (推广大使提现)
                 │
                 ▼
           PostgreSQL 16 (Docker volume pg-data)
```

## 3. 安全流 (请求经过的 7 层中间件链)

```
  请求进入 FastAPI
       │
       ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  ① CORSMiddleware                                                       │
  │     允许来源: *.liankebao.top                                          │
  │     允许方法: GET, POST, PUT, DELETE, OPTIONS                          │
  │     允许头: Authorization, Content-Type, X-Requested-With              │
  └─────────────────────────────────────────────────────────────────────────┘
       │
       ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  ② RequestLogMiddleware                                                 │
  │     记录: 时间, 方法, 路径, 状态码, IP, User-Agent, 响应时间(ms)         │
  │     敏感字段脱敏: password, token, secret                               │
  └─────────────────────────────────────────────────────────────────────────┘
       │
       ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  ③ RateLimitMiddleware (Redis 后端)                                     │
  │     策略:                                                               │
  │       - 匿名: 60 req/min per IP                                         │
  │       - 已认证: 300 req/min per user                                    │
  │       - 支付回调: 无限制 (白名单 IP)                                     │
  │       - 超出返回 429 Too Many Requests                                  │
  └─────────────────────────────────────────────────────────────────────────┘
       │
       ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  ④ AuthMiddleware (JWT)                                                 │
  │     公开路径放行: /health, /api/auth/login, /api/auth/register         │
  │     JWT Bearer Token 验证 → 解析 payload → 注入 request.user           │
  │     Token 过期 → 401 Unauthorized                                       │
  │     Token 无效/篡改 → 401 Unauthorized                                  │
  └─────────────────────────────────────────────────────────────────────────┘
       │
       ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  ⑤ RoleMiddleware                                                       │
  │     角色层级: admin > manager > user > guest                            │
  │     装饰器 @require_roles([\"admin\"]) 在 Router 层控制                  │
  │     越权访问 → 403 Forbidden                                            │
  └─────────────────────────────────────────────────────────────────────────┘
       │
       ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  ⑥ AuditMiddleware                                                     │
  │     关键操作记录: 创建/修改/删除 资源                                   │
  │     记录内容: who, what, when, ip, user_agent, diff                     │
  │     存储: audit_log 表 (PostgreSQL)                                     │
  └─────────────────────────────────────────────────────────────────────────┘
       │
       ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  ⑦ ExceptionMiddleware                                                  │
  │     捕获所有未处理异常                                                   │
  │     统一 JSON 格式: {"detail": "...", "code": "..."}                   │
  │     生产环境隐藏敏感错误栈 (500 → "内部服务器错误")                      │
  └─────────────────────────────────────────────────────────────────────────┘
       │
       ▼
     Router → Service → Model → Database
```

## 4. CI/CD 管道流

```
  ┌───────────────────────────────────────────────────────────────────────────┐
  │                     GitHub Actions 持续部署流水线                          │
  └───────────────────────────────────────────────────────────────────────────┘

  触发条件
     │
     ├─ push → main 分支 (自动部署)
     ├─ workflow_dispatch (手动触发, 可选跳过测试/构建)
     └─ concurrency: cd-prod (禁止并行部署)
       │
       ▼
  ┌──────────────────┐
  │  Job 1: test      │ ◀── 可跳过 (skip_tests=true)
  │  测试门禁          │
  └──────────────────┘
       │
       ├─ 检出代码
       ├─ Python 3.12 + pip cache
       ├─ pip install -r requirements.txt
       ├─ pytest (必须 100% 通过)
       ├─ coverage (阈值 >= 70%)
       └─ 失败 → 阻断部署 ❌
       │
       ▼
  ┌──────────────────┐
  │  Job 2: deploy    │ ◀── 依赖 test job
  │  构建 & 部署       │
  └──────────────────┘
       │
       ├─ [2.1] 检出代码
       ├─ [2.2] Node.js 20 + npm cache
       ├─ [2.3] 前端构建 (npm install && npm run build)
       ├─ [2.4] SSH 部署到阿里云 (重试 3 次)
       │         │
       │         ├─ 进入部署目录 /var/www/chainke-full
       │         ├─ git fetch + reset --hard (拉最新代码)
       │         ├─ pip install 依赖
       │         ├─ 配置 .env (如不存在)
       │         ├─ 重启服务 (优先 systemctl, 其次 supervisorctl, 最后 docker-compose)
       │         └─ 健康检查 (6 次重试, 5s 间隔)
       │         │
       │         └─ 失败 → 重试 (最多 3 次 SSH)
       │
       └─ [2.5] 清理 SSH 密钥
       │
       ▼
  ┌──────────────────┐
  │  Job 3: notify    │ ◀── 始终执行 (always())
  │  部署通知          │
  └──────────────────┘
       │
       ├─ 成功 → 飞书消息: 🟢 生产部署成功
       └─ 失败 → 飞书消息: 🔴 生产部署失败
       │
       ▼
  ┌───────────────────────────────────────────────────────────────────────────┐
  │                     蓝绿部署流程 (部署脚本)                                │
  └───────────────────────────────────────────────────────────────────────────┘

  blue_green_deploy.sh  (Docker Compose 方式)
       │
       ├─ [1/6] 前置检查 (docker, curl, compose 文件存在性)
       ├─ [2/6] 构建新版本 (docker compose build backend-{color})
       ├─ [3/6] 启动新环境容器 (docker compose up -d)
       ├─ [4/6] 健康检查 (curl http://127.0.0.1:{port}/health)
       │           blue  → port 8002
       │           green → port 8003
       ├─ [5/6] 切换 Nginx upstream
       │           cp upstream-{color}.conf → upstream-active.conf
       │           docker exec nginx -s reload
       └─ [6/6] 清理旧环境
                   docker compose rm -fsv backend-{old_color}

  状态文件: deploy/.blue-green-state  (记录当前活跃颜色)
  切换历史: deploy/.blue-green-history
```
