# 数据分级策略 — Data Classification Policy

> **项目**: AI 数智名片 (AI Digital Business Card)
> **生效日期**: 2026-07-01
> **策略所有者**: 安全与合规团队
> **适用范围**: 所有由 AI 数智名片 后端系统存储、处理、传输的数据

---

## 1. 分级体系总览

| 级别 | 标签 | 颜色 | 定义 | 示例 |
|------|------|------|------|------|
| **L1** | 🔓 **公开** | 绿色 | 可向公众完全公开的信息 | 产品介绍, 公开API文档, 公司名称 |
| **L2** | 🔐 **内部** | 蓝色 | 仅限内部员工/合作方访问 | 非PII业务统计, 内部API文档, 团队名称 |
| **L3** | 🛡️ **敏感** | 黄色 | 泄露可能对用户或业务造成中等影响 | 用户联系方式(脱敏后), 名片内容, 匹配记录, 支付记录 |
| **L4** | ⛔ **机密** | 红色 | 泄露可能造成严重/法律后果 | 明文手机号, 微信openid, 密码哈希, 支付密钥, JWT密钥 |

### 数据分级判断树

```
数据是否包含 PII？ ──是──→ 是否明文存储？ ──是──→ L4 机密
                          │                   └─否──→ L3 敏感
                          └─否──→ 数据泄露是否影响用户隐私或业务安全？
                                  ├─是──→ L3 敏感
                                  ├─内部运营需要──→ L2 内部
                                  └─公开无影响──→ L1 公开
```

## 3. 每级数据的保护要求

### L1 🔓 公开

| 控制维度 | 要求 |
|----------|------|
| **存储** | 无需加密 |
| **传输** | 建议 TLS (无强制要求) |
| **访问控制** | 无需认证 |
| **缓存** | 允许 CDN / 浏览器缓存 |
| **输出脱敏** | 无需脱敏 |
| **保留期限** | 永久 (合规要求除外) |
| **审计** | 无需审计 |
| **示例端点** | `/api/public/brochures?token=xxx`, `/health`, `/docs` |

### L2 🔐 内部

| 控制维度 | 要求 |
|----------|------|
| **存储** | 推荐使用数据库级别的访问控制, 明文存储可接受 |
| **传输** | ✅ 必须 TLS |
| **访问控制** | ✅ 需要认证; 建议 RBAC 限制读写 |
| **缓存** | 禁止 CDN 缓存; 内部缓存 TTL ≤ 300s |
| **输出脱敏** | 无需脱敏, 但仅限认证用户 |
| **保留期限** | 业务需要 + 最长 7 年 (税务/法律) |
| **审计** | 关键写操作审计 (CREATE / UPDATE / DELETE) |
| **示例** | `/api/brochures` (已认证), 团队成员列表 |

### L3 🛡️ 敏感

| 控制维度 | 要求 |
|----------|------|
| **存储** | ✅ 传输加密 + 推荐存储加密 (AES-256) |
| **传输** | ✅ 必须 TLS 1.2+ |
| **访问控制** | ✅ RBAC + 数据级权限 (仅本人或管理员) |
| **缓存** | ❌ 禁止缓存 (Cache-Control: no-store) |
| **输出脱敏** | ✅ 默认脱敏 (如 phone 显示 138****0000) |
| **保留期限** | 业务结束后 ≤ 90 天 (匿名化或删除) |
| **审计** | ✅ 所有访问和操作必须审计 |
| **日志脱敏** | ✅ 输出至日志时需脱敏 |
| **示例** | `/api/gdpr/data`, 名片内容, 匹配记录, 访客IP |

### L4 ⛔ 机密

| 控制维度 | 要求 |
|----------|------|
| **存储** | ✅ 必须加密 (AES-256-GCM), 独立密钥, 密钥轮换 ≤ 90 天 |
| **传输** | ✅ 必须 TLS 1.3 (TLS 1.2 最低接受) |
| **访问控制** | ✅ 最小权限原则: 仅极少数管理员可访问; 启用 MFA |
| **缓存** | ❌ 禁止任何形式的缓存 |
| **输出脱敏** | ✅ 绝对禁止未脱敏的输出 (API 响应必须 mask) |
| **保留期限** | 最短业务需要, 到期立即安全删除 (不可恢复) |
| **审计** | ✅ 每次访问必须审计, 包括失败的尝试 |
| **日志脱敏** | ✅ 严禁明文写入任何日志 (审计日志也需要脱敏) |
| **密钥管理** | ✅ 使用 KMS / 硬件安全模块 (HSM) 管理密钥 |
| **示例** | 明文手机号, wechat_openid, password_hash, JWT_SECRET, 支付密钥 |

## 4. 数据生命周期控制矩阵

| 阶段 | L1 公开 | L2 内部 | L3 敏感 | L4 机密 |
|------|---------|---------|---------|---------|
| **采集** | 无需同意 | 明确目的告知 | ✅ 用户明确同意 | ✅ 单独明示同意 |
| **存储** | 明文 | 明文/ACL | AES-256 加密 | AES-256-GCM + 独立密钥 |
| **访问** | 无限制 | 认证 + RBAC | 数据级 + RBAC | 最小权限 + MFA |
| **传输** | HTTP 可接受 | TLS 1.2+ | TLS 1.2+ | TLS 1.3 |
| **展示** | 明文 | 明文 (内部) | 脱敏 (默认) | 禁止展示 |
| **备份** | 明文 | 加密 | 加密 | 加密 + 独立密钥 |
| **销毁** | 常规删除 | 安全擦除 | 安全擦除 | 不可恢复销毁 |
| **审计** | 不审计 | CREATE/UPDATE/DELETE | 所有 CRUD | 每次访问 |

## 5. 数据脱敏规则

| 数据类型 | 脱敏规则 | 示例 |
|----------|----------|------|
| **手机号** | 显示前3后4, 中间4位用 `*` 替代 | `138****0000` |
| **姓名** | 保留姓氏, 名字用 `*` 替代 | `张*` / `李**` |
| **邮箱** | 用户名部分前2位 + `***` + 域名 | `te***@example.com` |
| **IP 地址** | 保留前3段, 最后一段用 `*` | `192.168.1.*` |
| **微信 openid** | 完全隐藏 (仅在需要时由系统内部使用) | `********` |
| **公司/职位** | 不脱敏 (L2 内部) | — |
| **头像 URL** | 不脱敏 | — |
| **地址** | 保留省份/城市, 详细地址用 `*` | `广东省深圳市***` |

## 6. 角色数据访问矩阵

| 角色 | L1 公开 | L2 内部 | L3 敏感 | L4 机密 |
|------|---------|---------|---------|---------|
| **未认证用户** | ✅ 可访问 | ❌ | ❌ | ❌ |
| **普通用户 (user)** | ✅ | ✅ 本人数据 | ✅ 本人数据(脱敏) | ❌ |
| **企业用户 (enterprise)** | ✅ | ✅ 本团队数据 | ✅ 本团队数据(脱敏) | ❌ |
| **管理员 (admin)** | ✅ | ✅ 全部 | ✅ 全部(脱敏) | ⚠️ 仅必要场景 |
| **超级管理员 (superadmin)** | ✅ | ✅ 全部 | ✅ 全部(脱敏) | ✅ 最小权限(MFA) |
| **审计员** | ✅ | ✅ 只读 | ✅ 只读(日志) | ✅ 只读(审计日志) |

## 7. 违规处理

| 违规行为 | 处理措施 |
|----------|----------|
| 开发环境使用生产数据 (包含 L3/L4) | 立即停止, 报告 CISO, 安全擦除 |
| 将 L4 数据明文写入日志 | 立即修复, 清理日志, 事故事件调查 |
| 未授权访问 L3/L4 数据 | 安全事件上报, 访问权限收回, 纪律处分 |
| 未脱敏展示 L4 数据 | 紧急修复, 数据泄露评估 |

## 8. 附录

### 8.1 数据分类标签模板

建议在代码中使用装饰器或注释标记数据分类:

```python
# 示例: Pydantic 模型中的数据分类标记
from pydantic import BaseModel, Field

class UserResponse(BaseModel):
    id: int = Field(description="用户ID", classification="L2-内部")
    phone: str = Field(description="手机号", classification="L4-机密")
    name: str = Field(description="姓名", classification="L3-敏感", mask="name")
    company: str = Field(description="公司", classification="L2-内部")
    membership_tier: str = Field(description="会员等级", classification="L2-内部")
```

| 级别 | 复审频率 | 触发条件 |
|------|----------|----------|
| L1 公开 | 每年 | 业务重大变更 |
| L2 内部 | 每年 | 团队/业务变更 |

| L4 机密 | 每季度 | 密钥轮换, 安全事件后立即 |
