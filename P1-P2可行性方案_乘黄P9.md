## 1. 联系人导入 (P1)

### Approach

**当前状态**: `graph/index.js` line 762-764, `importWechat()` 仅显示"功能开发中"占位。

**推荐方案**: **"分享链接+扫码+手动批量"三管齐下**

| 渠道 | 实现路径 | 优先级 |
|------|---------|--------|
| **分享邀请链接** | 利用微信小程序原生分享(onShareAppMessage)，用户A分享名片链接给微信好友B，B点击后自动建立联系关系 | P0 |
| **二维码扫码** | 复用已有扫码流程(scanQRCode)，扫描名片二维码直接建立连接 | P0 |
| **手动批量添加** | 改进现有 addContact()，支持批量输入姓名列表(每行一个) | P1 |
| **微信通讯录** | ❌ `wx.getPhoneNumber` 只能获取手机号不能读通讯录；`wx.chooseContact` 仅部分版本支持且需审核，**不建议走这条路** | 搁置 |

**影响页面**: `pages/network/graph/index.js`(主逻辑)、`pages/network/graph/index.wxml`(UI)

**关键考虑**:
- 微信小程序隐私接口审核趋严，通讯录读取不可行
- 推荐优先做"分享二维码→接收→建立联系"闭环
- 批量导入UI：文本区域输入姓名(每行一个)，循环调用 addContact() 或 API

## 2. 表单分步Stepper (P1)

### 四步Stepper方案

当前 `brochure/create/index.js` 是一个一页到底的长表单(~271行交互逻辑)，建议拆分为4步:

#### Step 1: 基本信息
- 头像(avatar)、姓名(name)、职位(title)、手机(phone)、邮箱(email)、微信(wechat)

#### Step 2: 专业信息
- 公司(company)、行业(industry)、公司规模(companySize)、公司简介(companyDesc)
- 发展历程(development)、公司图片(companyImages)
- 技能标签(skillTags)、个人简介(bio)

#### Step 3: 风格与目的
- 风格选择(style: professional/creative/minimal)
- 目的选择(purpose: partner/investor/employee/client)
- 资源供需(provides/needs)

#### Step 4: 预览与提交
- 展示完整画册预览效果
- 确认提交(submitForm)
- 草稿自动保存功能继续保留

**优势**:
- 每一步字段量合理(5-8个)，降低用户认知负担
- Step 1(必填:姓名/职位/公司)可快速完成，降低弃填率
- 第4步预览再确认，减少误提交

**实现注意**: 需新增 `currentStep` 状态变量，上一步/下一步导航，每一步独立验证。

## 3. 行业模板 (P2)

### 行业模板映射方案

当前 `formData.industry` 已存在，`brochureApi.getTemplate(purpose)` 已有接口。`TEST_BROCHURES` 中有3个样本(科技/金融/AI)。

构建模板映射表:

| 行业 | 推荐风格 | 推荐页面结构 | 特点 |
|------|---------|-------------|------|
| **科技/互联网** | minimal | Cover → Profile → Skills → Resources → Contact | 技能标签前置、案例页可选 |
| **金融/投资** | professional | Cover → Profile → Credentials → Company → Cases → Contact | 资质展示重要、公司介绍详细 |
| **教育/培训** | professional | Cover → Profile → Qualifications → Company → Contact | 学历/证书/资质页优先 |
| **医疗/健康** | professional | Cover → Profile → Certifications → Services → Contact | 资质认证前置、服务项目 |
| **制造/实业** | minimal | Cover → Profile → Company → Products → Cases → Contact | 产品和案例页重要 |

**实现方式**:
- 在 `pages/brochure/create/index.js` 新增 `industryTemplates` 配置对象
- 用户选择行业后，自动设置推荐风格(style)和页面结构顺序
- 利用已有的 `brochureApi.getTemplate(purpose)` + 行业参数

**当前状态**: `components/loading/index.wxml` 是一个简单的 spinner+text 组件。好消息是 `app.wxss` **已经存在 skeleton CSS** (lines 416-430)：

  border-radius: var(--radius-sm);

**方案**:

## 5. 字体大小可调 (P2)

### 方案

**当前**: `app.wxss` 使用 CSS 变量驱动所有字号(`--text-xs: 11px` ~ `--text-2xl: 32px`)，天然适合做字号缩放。

**实现**:
1. **Store 新增状态**: `store.js` 添加 `fontSize: 'medium'`，值为 'small' | 'medium' | 'large'
2. **字号缩放映射表**:

| 预设 | 缩放比例 | 效果 |
|------|---------|------|
| small | 0.9x | --text-base: 13.5px |
| medium | 1.0x | --text-base: 15px (当前) |
| large | 1.15x | --text-base: 17px |

3. **全局应用**: 在 `app.wxss` 的 `page` 上添加 data 属性选择器覆盖：

```css
page[data-font-size="small"] { --text-base: 13.5px; --text-sm: 11.5px; ... }
page[data-font-size="large"] { --text-base: 17px; --text-sm: 15px; ... }
```

4. **UI入口**: `pages/profile/profile.wxml` 设置菜单中新增"字体大小"选项，点击弹出 ActionSheet 选择小/中/大
5. **持久化**: 存入 Storage 键 'app_font_size'，onLaunch 时读取恢复

**优势**: 一次修改全站生效，无需逐个组件适配。仅需修改CSS变量。

## 6. 限次优化 (P2)

### 方案

**当前**: `pages/ai/scan/index.js` lines 49-88, `maxScans: 3`, 每月限制。`scanCount` 存于 `wx.getStorageSync('scanCount')`，无日期标记，无法区分月/日。

**推荐策略: "每天3次" 方案** (最简单、用户感知最好):

1. **存储改造**: 将 `scanCount` 改为复合存储:
   ```js
   wx.setStorageSync('scan_quota', { date: '2026-07-12', count: 2 })
   ```
2. **判断逻辑**:
   - onLoad 时读取 `scan_quota`
   - 比较 `scan_quota.date` 与 `new Date().toISOString().slice(0,10)`
   - 日期不同 → 重置 count=0，更新为今天
   - 日期相同 → 使用当前 count
3. **封顶显示**: count >= 3 时弹出升级引导(已有逻辑)
4. **后端配合**: 如果有多端限制，最终一致性需要服务端校验

**为什么选"每天3次"而非"首次完整体验后限次"**:
- 每天3次：实现简单、用户预期清晰、体验连续
- 首次完整体验后限次：需额外状态追踪，复杂度高

## 7. 匿名模式 (P2)

### 方案

**当前**: `store.js` 中 `userInfo` 包含 name/avatar/company/title，各页面直接展示真实信息。

**实现**:
1. **Store 新增状态**:
   ```js
   this._state = {
     ...existingState,
     anonymousMode: this._getStorage('app_anonymous', false),
   }
   ```
2. **新增方法** `setAnonymousMode(enabled)` — 设置状态 + 持久化
3. **显示替换规则**:

| 字段 | 匿名模式 ON |
|------|------------|
| name | → "匿名用户" |
| avatar | → 默认灰色头像 |
| company | → 隐藏 |
| title | → 隐藏 |
| 名片页(brochure) | 显示"对方已开启匿名" |

4. **UI入口**: `pages/profile/profile.wxml` 设置菜单中新增开关项"匿名模式"
5. **影响范围**:
   - `pages/profile/profile.wxml`: 用户信息头部
   - `pages/brochure/preview/index`: 名片预览页
   - `pages/card/card`: 我的名片tab
   - `pages/network/graph/index`: 图谱中的自节点
   - 分享时：所有对外展示的名片信息替换/隐藏

**关键考虑**:
- 匿名模式是**前端展示层**修改，后端依然存储真实信息
- 用户可随时关闭匿名模式恢复显示
- 需在隐私协议中说明匿名模式的限制(后台仍可追溯)
- 建议同时隐藏入脉网络中的个人信息
